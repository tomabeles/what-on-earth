# WOE-085: Decompose settings_content.dart Into Per-Section Files

**ID:** WOE-085
**Phase:** Quality -- Refactoring
**Depends on:** WOE-080
**Blocks:** WOE-082, WOE-083

---

## Description

`lib/shared/settings_content.dart` is an 873-line file containing ~15 widget classes: `SettingsBody`, the Display and Position Source sections, `_StaticPositionDialog` (~250 lines), coordinate fields, slider variants, and button styles. WOE-082 (Layers + Tile Cache sections) and WOE-083 (Account, Sensor, Power, About sections) will each add more sections to this file — split it **before** those land so they can be built as independent files.

Pure refactor: no behavior or visual changes. Existing tests (`settings_screen_test.dart`, `settings_display_test.dart`) must pass unmodified, or with import-path-only updates.

---

## Definition of Done

- [ ] `lib/shared/settings/` directory holds one file per concern:
  - [ ] `settings_body.dart` — top-level `SettingsBody` that composes sections
  - [ ] `display_section.dart`
  - [ ] `position_source_section.dart`
  - [ ] `static_position_dialog.dart`
  - [ ] `widgets.dart` (or similar) — shared coord fields, sliders, buttons used by more than one section
- [ ] `lib/shared/settings_content.dart` is deleted (or reduced to a re-export shim if external import churn is too large — prefer deletion)
- [ ] No widget code changes beyond moves and visibility adjustments (`_Foo` → `Foo` where cross-file access is needed)
- [ ] `flutter analyze` clean; all existing settings tests pass
