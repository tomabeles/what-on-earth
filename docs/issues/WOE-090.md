# WOE-090: CI Hardening -- Coverage Reporting and Format Check

**ID:** WOE-090
**Phase:** Infra -- CI
**Depends on:** none (WOE-088 adds the JS steps separately)

---

## Description

CI currently runs `flutter analyze`, `flutter test`, and debug builds for Android/iOS, but does not measure test coverage or enforce formatting. File-count coverage looks like ~75%; statement coverage is unknown, which makes it impossible to see whether the testing tickets (WOE-088, WOE-089, WOE-052) actually move the needle.

Small, self-contained workflow change.

---

## Definition of Done

- [ ] `flutter test --coverage` in the Linux CI job; `coverage/lcov.info` uploaded as a build artifact (Codecov optional — artifact is the minimum)
- [ ] Coverage summary printed in the job log (e.g. `lcov --summary` or a small Dart script) so the number is visible per-PR
- [ ] `dart format --set-exit-if-changed lib test` step before analyze (run `dart format` once across the repo in this ticket so the check starts green)
- [ ] Generated files (`*.g.dart`, `*.mocks.dart`) excluded from the coverage report
- [ ] CI total runtime increase < 1 minute

---

## Sub-tasks

- [ ] Add coverage flag + artifact upload to `.github/workflows/ci.yml`
- [ ] Add an lcov filter step removing `**/*.g.dart` and `**/*.mocks.dart`
- [ ] One-time `dart format .` commit (mechanical; separate commit so blame stays useful)
- [ ] Add the format check step
- [ ] Record the baseline coverage number in this ticket when it first runs
