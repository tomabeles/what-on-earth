# WOE-024: AR Compositing Cross-Platform Testing and Fixes

**ID:** WOE-024
**Phase:** 3 — Sensor Fusion and AR Camera
**Step:** 3.8
**Depends on:** WOE-023, WOE-005

---

## Description

Systematic testing of the AR compositing (camera feed + transparent globe) on both iOS and Android physical devices, with fixes for any platform-specific rendering issues discovered. This is a dedicated quality step — its output is a stable, confirmed working AR experience before proceeding to map layer work.

Common issues to look for: white flash on WebView init (WOE-005 addressed this, but verify), globe alpha blending artifacts, camera preview aspect ratio distortion, performance problems (dropped frames under compositing load).

**Reference:** TECH_SPEC §3.3

---

## Definition of Done

- [ ] AR compositing verified on at least one physical iOS device (iOS 16+): no visual artifacts
- [ ] AR compositing verified on at least one physical Android device (API 31+): no white flash, no opaque background
- [ ] Globe frame rate ≥ 30 fps during AR session (measure via CesiumJS `FRAME_RATE` bridge message)
- [ ] Camera preview has correct aspect ratio (no stretching or black bars)
- [ ] Moving the device rotates the globe correctly and smoothly
- [ ] All issues discovered are either fixed in this ticket or documented as follow-up issues with severity labels

---

## Sub-tasks

- [ ] Enable `FRAME_RATE` bridge message in CesiumJS: every 5 seconds, call `window.flutter_inappwebview.callHandler('FRAME_RATE', { fps: viewer.scene.frameRate })`
- [ ] Register `FRAME_RATE` handler in Flutter and log to console
- [ ] Test on iOS physical device:
  - Observe initial load for white flash (expect none after WOE-005 fixes)
  - Rotate device through 360°, verify globe tracks device orientation
  - Measure reported FPS (expect ≥ 30)
  - Check camera preview fills screen without distortion
- [ ] Test on Android physical device (same checklist)
- [ ] Fix any aspect ratio distortion: wrap `CameraPreview` in `AspectRatio(aspectRatio: controller.value.aspectRatio)` or use `FittedBox(fit: BoxFit.cover)` to ensure full-bleed
- [ ] If FPS < 30 on either platform: profile with Flutter DevTools and Xcode Instruments to identify bottleneck; common causes: bridge message frequency too high (throttle to 30 Hz), WebGL config, texture upload
- [ ] Document test results and any fixes applied in a brief test note committed with this issue
