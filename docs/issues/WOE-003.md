# WOE-003: Flutter Asset Declarations

**ID:** WOE-003
**Phase:** 1 — Foundation
**Step:** 1.3
**Depends on:** WOE-001, WOE-002

---

## Description

Declare the `assets/globe/` CesiumJS bundle and `assets/geodata/` Natural Earth GeoJSON directory in `pubspec.yaml` so that Flutter bundles them into the app. Also create the `assets/geodata/` directory with placeholder files so the asset declaration does not fail before real GeoJSON files are added in WOE-025.

**Reference:** TECH_SPEC §6.4

---

## Definition of Done

- [ ] `pubspec.yaml` flutter assets section includes `- assets/globe/` and `- assets/geodata/`
- [ ] `assets/geodata/` directory exists with a `.gitkeep` placeholder
- [ ] `flutter pub get` succeeds
- [ ] `flutter build apk --debug` succeeds (builds without the actual globe files since `assets/globe/` is git-ignored; CI will run `build_globe.sh` first)
- [ ] A note is added to the repo `README.md` explaining that `scripts/build_globe.sh` must be run before `flutter run`

---

## Sub-tasks

- [ ] Add the following to `pubspec.yaml` under `flutter:`:
  ```yaml
  assets:
    - assets/globe/
    - assets/geodata/
  ```
- [ ] Create `assets/geodata/.gitkeep`
- [ ] Verify `flutter pub get` completes cleanly
- [ ] Add a "Local development setup" section to `README.md` noting the `build_globe.sh` prerequisite
