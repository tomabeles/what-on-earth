# WOE-088: web_globe JS Test Harness (Vitest + ESLint + CI)

**ID:** WOE-088
**Phase:** Quality -- Testing
**Depends on:** none

---

## Description

`web_globe/src/` (~1,150 LOC: layer loading, GeoJSON label dedup, SGP4 propagation, pin rendering, reticle spatial lookup) has zero tests and no linting, while the Dart side of the same bridge contract is well tested. The most likely regression class in this app is Flutter and JS drifting out of contract, and nothing would catch it.

Add Vitest + ESLint to `web_globe/`, write a first tranche of unit tests, and run both in CI alongside the existing Vite build.

Includes two small code fixes found in review:
- `satellite_propagator.js`: `propagateNow()` returns `null` silently on SGP4 failure — add a `console.warn` (decayed/stale TLE should be visible in the WebView console, which Flutter already forwards to `debugPrint`).
- `layers.js`: the water-label deduplication block is duplicated three times (marine/lakes/rivers) — extract one helper, which the new tests then cover.
