# WOE-046: Onboarding Flow 3-Step PageView

**ID:** WOE-046
**Phase:** 6 — Cloud Sync and Onboarding
**Step:** 6.4
**Depends on:** WOE-032, WOE-048

---

## Description

Assemble the complete 3-step onboarding flow in a `PageView`. The three steps are:
1. Welcome / position source confirmation
2. Tile download (from WOE-032)
3. Magnetometer calibration (from WOE-047)

Steps are resumable — if the user closes the app mid-onboarding, they return to the last incomplete step.

**Reference:** TECH_SPEC §7.5, PRD FR-ONB-001 through FR-ONB-004

---

## Definition of Done

- [ ] `OnboardingFlow` widget wraps a `PageView` with 3 pages
- [ ] Page 1 (Welcome): app logo, tagline, "This app uses your device sensors and the ISS position feed to show you a live view of Earth. No setup needed." CTA: "Get Started →"
- [ ] Page 2 (Tile Download): embeds `TileDownloadStep` from WOE-032; "Next →" enabled after download completes or user taps "Skip"
- [ ] Page 3 (Calibration): embeds calibration screen from WOE-047; "Done" or "Skip" button
- [ ] Completing step 3 (or skipping): marks onboarding complete in shared preferences and navigates to `ARView`
- [ ] Onboarding is shown on first launch; skipped on subsequent launches
- [ ] Per PRD FR-ONB-004: the AR view starts loading in the background during onboarding (the globe initializes but is not visible until onboarding completes)
- [ ] Widget tests for page transitions and completion flag

---

## Sub-tasks

- [ ] Create `lib/onboarding/onboarding_flow.dart`
- [ ] Use a `PageController` with `physics: NeverScrollableScrollPhysics()` (navigation via buttons, not swipe, to prevent accidental skipping)
- [ ] Page 1: `WelcomeStep` widget with app branding and position source explanation; "Get Started" button advances to page 2
- [ ] Page 2: embed `TileDownloadStep`; "Next" button advances to page 3 (active after download or skip)
- [ ] Page 3: embed `CalibrationStep` from WOE-047; "Done" / "Skip" button completes onboarding
- [ ] On completion: call `OnboardingStateManager.markComplete()` (WOE-048) then `Navigator.pushReplacement(ARScreen)`
- [ ] In `main.dart` / `app.dart`: check `OnboardingStateManager.isComplete()` on launch; route to `OnboardingFlow` or `ARView` accordingly
- [ ] Widget test: verify page 1 → 2 transition on button tap; verify completion flag is set
