# WOE-037: Add Pin Bottom Sheet UI

**ID:** WOE-037
**Phase:** 5 — Pins
**Step:** 5.4
**Depends on:** WOE-036, WOE-035

---

## Description

Implement the full `AddPinSheet` bottom sheet widget. When a user taps the globe, this sheet slides up and allows them to name the pin, add an optional note, and choose an icon from five options. On confirm, the pin is saved via `PinRepository`.

**Reference:** TECH_SPEC §7.4, PRD FR-PIN-001, FR-PIN-002

---

## Definition of Done

- [ ] `AddPinSheet` widget exists in `lib/pins/`
- [ ] Shows the tapped coordinates (formatted as `°N/S °E/W`) in a read-only location field
- [ ] Name text field (required, max 100 characters, keyboard focus on open)
- [ ] Note text field (optional, max 500 characters, multi-line)
- [ ] Icon picker: 5 icons displayed as selectable chips; default icon 0 selected
- [ ] "Add Pin" button is disabled until name has at least 1 character
- [ ] Tapping "Add Pin" calls `PinRepository.createPin(...)` and dismisses the sheet
- [ ] Tapping outside or "Cancel" dismisses without saving
- [ ] Widget tests for validation, icon selection, and save action

---

## Sub-tasks

- [ ] Create `lib/pins/add_pin_sheet.dart`
- [ ] Use `DraggableScrollableSheet` or `showModalBottomSheet` with `isScrollControlled: true`
- [ ] Icon assets: use Material Icons for the 5 pin icons: `Icons.home`, `Icons.star`, `Icons.favorite`, `Icons.place`, `Icons.flag` — displayed as a `Wrap` of `ChoiceChip` widgets
- [ ] Use a `Form` with `TextFormField` for name (validator: not empty) and note
- [ ] "Add Pin" button: `ElevatedButton` enabled when `formKey.currentState.validate()` is true
- [ ] On "Add Pin" tap: call `ref.read(pinRepositoryProvider).createPin(name: ..., latDeg: ..., lonDeg: ..., iconId: ..., note: ...)` then `Navigator.pop(context)`
- [ ] Widget test: verify button disabled when name empty; verify `PinRepository.createPin` called with correct args on submit
