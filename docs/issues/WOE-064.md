# WOE-064: Theme Riverpod Provider + Persistence

**ID:** WOE-064
**Phase:** UI -- Theme Foundation
**Depends on:** WOE-063
**UI Spec:** SS2.1

---

## Description

Create a Riverpod provider that manages the active theme ID and persists the user's choice in SharedPreferences under the key `ui_theme_id`. When the theme changes, `ThemeData` in `app.dart` rebuilds with the new `AppTokens`. Default theme on first launch: `night`.

---

## Definition of Done

- [ ] `themeProvider` (or `@riverpod` annotated) exposes the active `AppTheme`
- [ ] `setTheme(String id)` method persists the ID to SharedPreferences and updates state
- [ ] On app launch, reads `ui_theme_id` from SharedPreferences; falls back to `night`
- [ ] `app.dart` watches the provider and rebuilds `MaterialApp` theme on change
- [ ] Changing theme immediately updates all themed widgets
- [ ] Unit test: set theme to `dark`, restart provider, verify `dark` is restored

---

## Sub-tasks

- [ ] Create `lib/shared/theme_provider.dart`
- [ ] Define provider using `@riverpod` annotation (or manual `StateNotifier`)
- [ ] `build()`: read `ui_theme_id` from SharedPreferences, return `AppThemeRegistry.find(id)`
- [ ] `setTheme(String id)`: write to SharedPreferences, update state
- [ ] Update `app.dart`: wrap `MaterialApp` in `Consumer`, watch `themeProvider`, build `ThemeData` with `extensions: [theme.tokens]` and `scaffoldBackgroundColor: theme.tokens.surfacePrimary`
- [ ] Run `dart run build_runner build` if using `@riverpod`
- [ ] Test: verify default is `night`; verify persistence round-trip
