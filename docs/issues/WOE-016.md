# WOE-016: StaticSource for Training Mode

**ID:** WOE-016
**Phase:** 2 — Position Module
**Step:** 2.9
**Depends on:** WOE-008, WOE-013

---

## Description

Implement `StaticSource`, the third concrete `PositionSource` that emits a fixed lat/lon/alt on a 10-second interval. This is used for ground-based training scenarios and simulator testing. The static coordinates are configured via the Settings screen (WOE-049) and stored in shared preferences.

**Reference:** TECH_SPEC §7.1, PRD FR-POS-001 (Static/TLE Propagation mode)

---

## Definition of Done

- [ ] `StaticSource` implements `PositionSource`
- [ ] Emits the configured lat/lon/alt every 10 seconds with `sourceType: static`
- [ ] Coordinates are settable via constructor and read from shared preferences by default
- [ ] `PositionController` can be switched to `StaticSource` via `setSourceMode(PositionSourceType.static)` with provided coordinates
- [ ] Unit tests: verifies stream emits the correct static position

---

## Sub-tasks

- [ ] Create `lib/position/static_source.dart`
- [ ] Constructor accepts `double latDeg`, `double lonDeg`, `double altKm` and optional `Duration interval` (default 10 seconds)
- [ ] `start()`: emit one position immediately, then `Timer.periodic`
- [ ] `stop()`: cancel timer
- [ ] Add a `staticCoordinatesProvider` that reads from shared preferences: `static_lat`, `static_lon`, `static_alt_km`
- [ ] Update `PositionController.setSourceMode()` to handle `PositionSourceType.static`: stop current source, create `StaticSource` with stored coordinates, start it
- [ ] Unit test in `test/unit/static_source_test.dart`
