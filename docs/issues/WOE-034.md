# WOE-034: Drift Database Schema Generation (Pins and TileMetadata)

**ID:** WOE-034
**Phase:** 5 — Pins
**Step:** 5.1
**Depends on:** WOE-001

---

## Description

Define the drift table schemas for `Pins` and `TileMetadata`, run code generation, and verify the generated DAOs and database accessors compile correctly. This is the foundation for all pin and tile cache persistence.

**Reference:** TECH_SPEC §7.4, §9.3, §9.4

---

## Definition of Done

- [ ] `Pins` table defined in drift with all columns from TECH_SPEC §9.3
- [ ] `TileMetadata` table defined in drift with all columns from TECH_SPEC §9.4
- [ ] `build_runner` code generation completes without errors
- [ ] Both tables are accessible via a `AppDatabase` class with typed DAOs
- [ ] `flutter analyze` passes on all generated and hand-written files
- [ ] A `test/unit/database_test.dart` verifies that the database opens and both tables can be created on an in-memory SQLite instance

---

## Sub-tasks

- [ ] Create `lib/pins/pin_database.dart`:
  - Define `class Pins extends Table` with columns: `id` (text, PK), `latDeg` (real), `lonDeg` (real), `name` (text, max 100), `note` (text, nullable), `iconId` (int, default 0), `createdAt` (datetime), `updatedAt` (datetime), `deletedAt` (datetime, nullable)
  - Define `class PinsDao` with `@DriftAccessor(tables: [Pins])`: methods for insert, select all non-deleted, update, soft-delete
- [ ] Create `lib/tile_cache/tile_database.dart`:
  - Define `class TileMetadataTable extends Table` with columns matching TECH_SPEC §9.4
  - Define `class TileDao`
- [ ] Create `lib/shared/app_database.dart`:
  - `@DriftDatabase(tables: [Pins, TileMetadataTable])` annotated `AppDatabase` class
  - Include both DAOs
  - Use `FlutterQueryExecutor.inDatabaseFolder(path: 'app.db')`
- [ ] Run `dart run build_runner build --delete-conflicting-outputs`
- [ ] Create `appDatabaseProvider` Riverpod provider (lazy singleton)
- [ ] Write `test/unit/database_test.dart` using an in-memory `NativeDatabase.memory()` database
