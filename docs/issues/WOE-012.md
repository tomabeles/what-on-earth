# WOE-012: TLESource Dart Class

**ID:** WOE-012
**Phase:** 2 — Position Module
**Step:** 2.5
**Depends on:** WOE-010, WOE-011

---

## Description

Implement the `TLESource` concrete `PositionSource` that drives SGP4 position propagation via the bridge. On `start()`, it loads the stored TLE, sends it to the WebView via `SET_TLE`, and subscribes to `POSITION_UPDATE` messages from the bridge. All emitted positions have `sourceType: estimated`.

**Reference:** TECH_SPEC §7.1

---

## Definition of Done

- [ ] `TLESource` implements `PositionSource`
- [ ] On `start()`: loads stored TLE via `TleManager.loadStored()`; if TLE is available, sends `SET_TLE` via bridge; subscribes to `BridgeController.propagatedPositions`
- [ ] If no stored TLE is available, `TLESource` logs a warning and the stream stays silent
- [ ] On `stop()`: cancels the bridge subscription; does not send a stop message to JS (interval will be superseded by next `SET_TLE`)
- [ ] All emitted positions have `sourceType: PositionSourceType.estimated`
- [ ] When `TleManager.fetchAndStore()` completes (TLE refresh), `TLESource` sends the updated TLE to the WebView via bridge automatically
- [ ] Unit test: mock bridge and verify `SET_TLE` is sent on start with correct TLE lines

---

## Sub-tasks

- [ ] Create `lib/position/tle_source.dart`
- [ ] Constructor takes `TleManager` and `BridgeController` as dependencies
- [ ] `start()`: load TLE → if null, log warning and return; send `SET_TLE` bridge message with `line1` and `line2` parsed from the TLE string; subscribe to `BridgeController.propagatedPositions` and forward to internal `StreamController`
- [ ] Implement TLE refresh listener: when `TleManager` successfully fetches a new TLE (expose a `Stream<String>` from `TleManager` that emits the new TLE string), re-send `SET_TLE` to WebView
- [ ] Parse TLE string: split on newlines; line 0 is name, line 1 starts with `1 `, line 2 starts with `2 `
- [ ] Write unit test verifying `SET_TLE` message content
