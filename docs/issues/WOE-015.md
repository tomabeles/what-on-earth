# WOE-015: Position Source Status Indicator Widget

**ID:** WOE-015
**Phase:** 2 — Position Module
**Step:** 2.8
**Depends on:** WOE-013

---

## Description

Implement the persistent status bar widget that displays the current position source, data freshness, and connectivity state. This widget appears on all primary screens (AR view, 2D map) and is always visible without requiring navigation.

Per TECH_SPEC §10.3 and PRD FR-POS-005, the widget must show: source type, live vs. estimated, age of last live fix, and an online/offline indicator.

**Reference:** TECH_SPEC §10.3, PRD §6.1 FR-POS-005

---

## Definition of Done

- [ ] `StatusBar` widget exists in `lib/shared/status_bar.dart`
- [ ] Displays source label: "ISS Live" (green), "Estimated (TLE)" (amber), or "Static" (grey)
- [ ] Displays age of last fix, formatted as "Updated Xs ago" — refreshes every second
- [ ] Displays a connectivity icon (green dot = online, grey dot = offline) using `connectivity_plus`
- [ ] Widget rebuilds reactively when `PositionController` status changes
- [ ] Text and icons meet WCAG 2.1 AA contrast on the dark globe background
- [ ] Widget is compact (fits in a 40dp high strip at the top of the screen)

---

## Sub-tasks

- [ ] Create `lib/shared/status_bar.dart`
- [ ] Use `ref.watch(positionControllerProvider)` to get `PositionSourceStatus`
- [ ] Display a colored dot + label for source type (green/amber/grey)
- [ ] Display a `StreamBuilder` or `Timer`-driven "X seconds ago" counter using `positionSourceStatus.lastFixAt`
- [ ] Use `connectivity_plus`'s `ConnectivityResult` stream for the online/offline dot
- [ ] Widget test: verify correct label and color for each `PositionSourceType`
