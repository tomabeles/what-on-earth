# WOE-048: Onboarding State Persistence and Completion Banner

**ID:** WOE-048
**Phase:** 6 — Cloud Sync and Onboarding
**Step:** 6.6
**Depends on:** WOE-001

---

## Description

Implement `OnboardingStateManager` that persists the completion state of each onboarding step as a bitmask in shared preferences, and a non-blocking banner on the AR view that prompts users to complete setup if any step is incomplete.

Per PRD FR-ONB-003 and FR-ONB-004, onboarding is resumable and the app is usable before it is complete.

**Reference:** TECH_SPEC §7.5, PRD FR-ONB-003, FR-ONB-004

---

## Definition of Done

- [ ] `OnboardingStateManager` class exists in `lib/onboarding/`
- [ ] Bitmask stored in shared preferences key `onboarding_complete_mask`
- [ ] Bit 0 = welcome complete; bit 1 = tile download complete; bit 2 = calibration complete
- [ ] `isComplete()` returns true if all three bits are set
- [ ] `markStepComplete(int stepIndex)` sets the corresponding bit
- [ ] `getCurrentStep()` returns the index of the first incomplete step (for resuming)
- [ ] `OnboardingBanner` widget shown on `ARView` when `!isComplete()`: "Finish setup →" dismissible banner
- [ ] Tapping banner navigates to `OnboardingFlow` at the correct step
- [ ] Banner does not show once `isComplete()` is true

---

## Sub-tasks

- [ ] Create `lib/onboarding/onboarding_state_manager.dart`
- [ ] Use `SharedPreferences` to read/write the `onboarding_complete_mask` integer
- [ ] Implement `bool isComplete()`: `(mask & 0x7) == 0x7` (all 3 bits set)
- [ ] Implement `void markStepComplete(int step)`: `mask |= (1 << step)`, write back
- [ ] Implement `int getCurrentStep()`: return first bit index that is 0
- [ ] Create `onboardingStateProvider` as a `StateNotifierProvider<OnboardingStateManager, int>` (state = current mask)
- [ ] Create `lib/onboarding/onboarding_banner.dart`: `AnimatedContainer` that appears at the top of the AR view; shown when `!isComplete()`; has a "Finish setup →" button and a dismiss (X) button
- [ ] Dismiss only for the current session — banner reappears on next app launch if still incomplete
- [ ] Add `OnboardingBanner` to the `ARView` Stack (layer 3, above the status bar)
- [ ] Widget test: banner is shown when mask is incomplete; hidden when mask = 0x7
