# WOE-066: Screen Navigation Routing

**ID:** WOE-066
**Phase:** UI -- AR Chrome
**Depends on:** WOE-065
**UI Spec:** SS3.1

---

## Description

Wire the speed-dial NAV FAB destinations to push actual screen routes. Create placeholder screen shells for 2D Map, Pin List, and Settings. Each placeholder shows a centered title and has the NAV FAB with the correct active-destination highlight. System back gesture returns to AR.

---

## Definition of Done

- [ ] Tapping "Map" in NAV FAB pushes `MapScreen` (placeholder)
- [ ] Tapping "Pins" pushes `PinListScreen` (placeholder)
- [ ] Tapping "Settings" pushes `SettingsScreen` (placeholder)
- [ ] System back button / swipe returns to AR View
- [ ] On each destination screen, NAV FAB shows with matching button highlighted
- [ ] AR View itself has no active-destination highlight on the FAB
- [ ] Navigation uses `Navigator.push` (not replacement -- AR View stays in stack)

---

## Sub-tasks

- [ ] Create placeholder `lib/screens/map_screen.dart`: `Scaffold` with `surfacePrimary` bg, centered "2D Map" text, `NavSpeedDial` with `activeDestination: 'map'`
- [ ] Create placeholder `lib/screens/pin_list_screen.dart` (if not already existing): same pattern, title "Pins"
- [ ] Create placeholder `lib/screens/settings_screen.dart` (if not already existing): same pattern, title "Settings"
- [ ] In `lib/screens/ar_screen.dart`: integrate `NavSpeedDial` in bottom-right, wire callbacks to `Navigator.push(MaterialPageRoute(...))`
- [ ] Ensure FAB collapses before navigation starts (call collapse, then navigate)
- [ ] Widget test: tap Map FAB item, verify MapScreen is pushed
