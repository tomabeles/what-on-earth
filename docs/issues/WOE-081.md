# WOE-081: Settings -- Position Source Section

**ID:** WOE-081
**Phase:** UI -- Settings
**Depends on:** WOE-013
**UI Spec:** SS4.6

---

## Description

Add the Position Source section to the Settings screen. Provides a segmented control (or radio group) to switch between Live ISS Feed, TLE Propagation, and Static mode. When Static is selected, lat/lon/alt entry fields appear.

---

## Definition of Done

- [ ] Position Source section card added to Settings `ListView`
- [ ] Segmented control or radio group with 3 options: Live ISS Feed, TLE Propagation, Static
- [ ] Current selection reflects `PositionController` active source type
- [ ] Changing selection calls `PositionController.setSourceMode(...)`
- [ ] When Static selected: lat, lon, alt `TextFormField`s appear below with numeric keyboards
- [ ] Static coordinate values persisted in SharedPreferences (`static_lat`, `static_lon`, `static_alt_km`)
- [ ] Input validation: lat -90 to 90, lon -180 to 180, alt 0 to 1000
- [ ] Widget test: segmented control toggles; static fields appear when static selected

---

## Sub-tasks

- [ ] Add Position Source section card to `SettingsScreen` ListView
- [ ] Use `SegmentedButton<PositionSourceType>` (Material 3) or `CupertinoSegmentedControl` with 3 segments
- [ ] Watch `positionControllerProvider` for current source type
- [ ] On segment change: call `ref.read(positionControllerProvider.notifier).setSourceMode(type)` with optional coordinates for static
- [ ] Conditional static fields: `AnimatedCrossFade` or `Visibility` showing 3 `TextFormField`s when `type == static`
- [ ] Each field: `TextFormField` with `keyboardType: TextInputType.numberWithOptions(decimal: true, signed: true)`
- [ ] Read default values from SharedPreferences; write on field blur or submit
- [ ] Validation: `FormField` validators for range constraints
- [ ] Widget test: select "Static", verify fields appear; select "Live", verify fields hidden
