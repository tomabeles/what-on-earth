# WOE-052: Offline Reliability Integration Tests

**ID:** WOE-052
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.3
**Depends on:** WOE-013, WOE-031, WOE-038

---

## Description

Write and run integration tests that verify all P0 features function correctly with zero network connectivity. These tests simulate a 30-minute communication blackout and verify that the app does not crash, display errors, or degrade core functionality.

**Reference:** PRD §7.3 NFR-REL-001, NFR-REL-002

---

## Definition of Done

- [ ] Integration test suite in `integration_test/offline_test.dart` exists and passes
- [ ] Tests verify: AR globe renders with TLE-propagated position when live API is unreachable
- [ ] Tests verify: GeoJSON vector layers render from local assets (no network required)
- [ ] Tests verify: raster tile layers render from pre-downloaded cache (tile server serves them locally)
- [ ] Tests verify: pins are readable and pass times are calculated when offline
- [ ] Tests verify: no crash or error screen appears when `connectivity_plus` reports offline
- [ ] Tests verify: `StatusBar` shows "Estimated (TLE)" source type when offline
- [ ] Manual test steps documented for verifying behavior on a real device in airplane mode

---

## Sub-tasks

- [ ] Create `integration_test/offline_test.dart` using `flutter_test` and `integration_test` packages
- [ ] Test setup: add a mock for `Dio` that always throws `DioException(type: DioExceptionType.connectionError)` to simulate network unavailability for the ISS API and TLE endpoints; use pre-seeded local TLE file and tile cache
- [ ] Test: launch app, wait for `GLOBE_READY`, verify `StatusBar` shows `PositionSourceType.estimated` within 15 seconds of simulated connectivity loss
- [ ] Test: verify globe is visible (find `GlobeView` widget, `isVisible()`)
- [ ] Test: verify creating a pin while offline works (create pin, verify it appears in `PinRepository.watchAllPins()`)
- [ ] Test: verify pass calculation works offline (uses local TLE)
- [ ] Write manual airplane mode test checklist in `docs/offline_test_checklist.md`:
  - Enable airplane mode before launching the app
  - Verify globe appears and moves
  - Verify status bar shows "Estimated (TLE)"
  - Verify existing pins are visible
  - Verify adding a new pin works
  - Verify pass time shown for a pin
