# WOE-038: SYNC_PINS Bridge and CesiumJS Pin Entities (pins.js)

**ID:** WOE-038
**Phase:** 5 — Pins
**Step:** 5.5
**Depends on:** WOE-035, WOE-006

---

## Description

Implement the `SYNC_PINS` Flutter→JS bridge message and `pins.js` in the CesiumJS bundle that renders personal pins as billboard entities on the globe at their correct geographic coordinates.

When pins change (create, update, delete), Flutter sends a full pin list to CesiumJS, which clears and re-renders all entities. This full-re-render approach is acceptable for V1 (expected < 50 pins).

**Reference:** TECH_SPEC §7.4, §8.1

---

## Definition of Done

- [ ] `SYNC_PINS` bridge message accepts `{ pins: [{ id, lat, lon, name, iconId }] }`
- [ ] CesiumJS removes all existing pin entities and re-renders the current list
- [ ] Pins render as billboard entities at the correct geographic position (on the globe surface, not floating above it)
- [ ] Each pin shows a distinct icon based on `iconId` (5 icon variants as small colored markers)
- [ ] Pin labels show the pin name, readable at orbital zoom
- [ ] Flutter watches `PinRepository.watchAllPins()` and sends `SYNC_PINS` whenever the list changes
- [ ] Widget test: verify `SYNC_PINS` is called when `PinRepository` emits a new list

---

## Sub-tasks

- [ ] Create `web_globe/src/pins.js` with:
  - `const pinDataSource = new Cesium.CustomDataSource('pins')` added to `viewer.dataSources`
  - `function syncPins(pinsArray)`: clear `pinDataSource.entities`; for each pin, add a `Cesium.Entity` with `position: Cesium.Cartesian3.fromDegrees(pin.lon, pin.lat)`, a `billboard` component (icon based on `iconId`), and a `label` component (pin name)
- [ ] Define 5 icon colors/images as CSS colored SVG data URIs or canvas-drawn icons: id 0=white, 1=amber, 2=red, 3=green, 4=blue (simple circle markers for V1)
- [ ] In `main.js`, handle `SYNC_PINS` message: call `syncPins(payload.pins)`
- [ ] In Flutter, in `GlobeView` or a `PinSyncController`, watch `pinRepositoryProvider.watchAllPins()` and on each emission call `bridgeController.send(OutboundMessage.syncPins, { 'pins': pins.map((p) => p.toJson()).toList() })`
- [ ] Only send `SYNC_PINS` after `GLOBE_READY` has been received
- [ ] Test on device: create a pin, verify it appears on the globe immediately
