# WOE-010: TLE Download, File Storage, and Refresh Daemon

**ID:** WOE-010
**Phase:** 2 — Position Module
**Step:** 2.3
**Depends on:** WOE-008

---

## Description

Implement TLE file management: downloading the current ISS TLE from CelesTrak, persisting it to the app documents directory, and refreshing it every 6 hours via a `flutter_background_fetch` periodic task. The stored TLE is the fallback used by `TLESource` when the live API is unavailable.

**Reference:** TECH_SPEC §4.2, §7.1

---

## Definition of Done

- [ ] `TleManager` class exists in `lib/position/tle_manager.dart`
- [ ] On `fetchAndStore()`, downloads from `https://celestrak.org/NORAD/elements/gp.php?CATNR=25544&FORMAT=TLE` and writes to `{documents}/tle/iss_latest.tle` along with a `{documents}/tle/iss_latest_timestamp.txt` file containing the ISO 8601 fetch time
- [ ] `loadStored()` reads and returns the stored TLE string; returns null if no file exists
- [ ] `TleRefreshDaemon` registers a `flutter_background_fetch` periodic task with 6-hour interval that calls `fetchAndStore()`
- [ ] Unit tests: successful fetch, network failure (does not overwrite existing file), load when file absent returns null

---

## Sub-tasks

- [ ] Create `lib/position/tle_manager.dart` with `TleManager` class
- [ ] Implement `Future<void> fetchAndStore()`: GET request to CelesTrak TLE endpoint; on success parse the 3-line response; write to `tle/iss_latest.tle`; write timestamp to `tle/iss_latest_timestamp.txt`; on failure log error and do not overwrite existing file
- [ ] Implement `Future<String?> loadStored()`: read `tle/iss_latest.tle`; return null if not found
- [ ] Implement `Future<DateTime?> lastFetchTime()`: read `tle/iss_latest_timestamp.txt`; parse and return
- [ ] Create `lib/position/tle_refresh_daemon.dart` with `TleRefreshDaemon.init()`:
  - Call `BackgroundFetch.configure(BackgroundFetchConfig(minimumFetchInterval: 360, ...))` (6 hours = 360 minutes)
  - In the `onFetch` callback, call `TleManager.fetchAndStore()` then `BackgroundFetch.finish(taskId)`
- [ ] Call `TleRefreshDaemon.init()` at app startup from `main.dart`
- [ ] Unit tests in `test/unit/tle_manager_test.dart` with a temp directory and mocked `Dio`
