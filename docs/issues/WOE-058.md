# WOE-058: Error Monitoring Integration (Opt-In)

**ID:** WOE-058
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.9
**Depends on:** WOE-053

---

## Description

Integrate Sentry (or equivalent) for crash reporting and error monitoring. Per PRD NFR-SEC-003, telemetry and analytics require explicit user opt-in. Error monitoring must be disabled by default and only activated after the user explicitly enables it in Settings.

The crash-free session rate target is ≥ 99.5% (PRD §7.3 NFR-REL-003).

**Reference:** PRD §7.3 NFR-REL-003, NFR-SEC-003

---

## Definition of Done

- [ ] `sentry_flutter` package added to `pubspec.yaml`
- [ ] Sentry is initialized in `main.dart` only if user has opted in (checked before `runApp`)
- [ ] A "Share crash reports" toggle is present in Settings (default: OFF)
- [ ] When the toggle is turned ON: Sentry is initialized for the current session; preference is stored; future sessions auto-initialize Sentry
- [ ] When the toggle is turned OFF: Sentry is not initialized; stored preference cleared
- [ ] Sentry DSN is injected via `--dart-define` (key: `SENTRY_DSN`); not hardcoded
- [ ] All uncaught Flutter and Dart exceptions are captured when opt-in is active
- [ ] No user-identifiable information (email, pin names, coordinates) is included in Sentry payloads

---

## Sub-tasks

- [ ] Add `sentry_flutter: ^8.x` to `pubspec.yaml`
- [ ] Add `SENTRY_DSN` to `--dart-define` variables and `.env.example`
- [ ] Create `lib/shared/error_monitoring.dart`:
  - `ErrorMonitoring.init(dsn)`: calls `SentryFlutter.init((options) { options.dsn = dsn; options.sendDefaultPii = false; }, appRunner: runApp)`
  - `ErrorMonitoring.isEnabled()`: reads `shared_preferences` key `error_monitoring_opt_in`
- [ ] In `main.dart`: read opt-in preference; if true, call `ErrorMonitoring.init(dsn)`, else call `runApp` directly
- [ ] Add "Share crash reports" `SwitchListTile` in `SettingsScreen` under an "About" or "Privacy" section; on toggle ON: store preference + show snackbar "Crash reporting enabled — takes effect on next launch"; on toggle OFF: store preference cleared + show "Crash reporting disabled"
- [ ] Before Sentry sends any event: add a `beforeSend` hook that strips any event fields containing known PII patterns (e.g., scrub `lat`, `lon`, `name` fields from breadcrumbs)
- [ ] Create a test release build and verify Sentry receives a test event when opt-in is active
