# WOE-032: Onboarding Tile Download UI Integration

**ID:** WOE-032
**Phase:** 4 — Map Layers
**Step:** 4.8
**Depends on:** WOE-030, WOE-031

---

## Description

Implement onboarding step 2: the tile download screen that allows users to select which layers to pre-download and shows download progress. This is the primary offline preparation step — after completing it, the app functions fully during communication blackouts.

The full onboarding flow (steps 1, 2, 3 in sequence with a `PageView`) is implemented in WOE-046. This issue implements step 2 as a standalone widget that WOE-046 will embed.

**Reference:** TECH_SPEC §7.5, PRD FR-ONB-002

---

## Definition of Done

- [ ] `TileDownloadStep` widget exists in `lib/onboarding/`
- [ ] Shows checkboxes for each downloadable layer: base map, relief shading, cloud cover
- [ ] Shows estimated download size per layer and total selected size
- [ ] "Download" button starts the download; shows a linear progress bar with `X of Y tiles (N MB)` label
- [ ] Download can be paused/cancelled
- [ ] "Skip" button is available with a warning snackbar: "App may not work fully during connectivity blackouts"
- [ ] On completion, shows a "Download complete" confirmation with tile counts per layer
- [ ] Widget tests for layer selection and progress display

---

## Sub-tasks

- [ ] Create `lib/onboarding/tile_download_step.dart`
- [ ] Define `LayerOption` model: `{layerId, displayName, estimatedSizeMb, sourceUrlTemplate, fileExtension}`
- [ ] Hardcode layer options (base: ~1.5 GB estimate; relief: ~300 MB; clouds: ~200 MB at zoom 0–5)
- [ ] Layer selection state: `Map<String, bool>` tracked in `StateNotifier`
- [ ] "Download" button: for each selected layer, call `TileDownloader.downloadLayer(...)` sequentially; combine progress streams
- [ ] Display `LinearProgressIndicator` with `value = progress.completedTiles / progress.totalTiles`
- [ ] Display secondary text: `'${progress.completedTiles}/${progress.totalTiles} tiles (${(progress.bytesDownloaded / 1e6).toStringAsFixed(1)} MB)'`
- [ ] Cancel button: call `cancelToken.cancel()` on the active download
- [ ] On success: store completed layer IDs in shared preferences (`downloaded_layers`); show completion UI
- [ ] Widget test: verify progress indicator updates when `TileDownloadProgress` stream emits
