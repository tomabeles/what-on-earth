# WOE-041: In-App Pass Approach Notification

**ID:** WOE-041
**Phase:** 5 — Pins
**Step:** 5.8
**Depends on:** WOE-040

---

## Description

Implement the in-app notification that alerts the crew member when the station is approaching viewing distance of a pinned location. A `Timer.periodic` running every 60 seconds checks whether any pin pass starts within a configurable lead time (default 5 minutes) and shows a banner if so.

Per PRD FR-PIN-006 and NFR-SEC-004, this is an in-app notification only — no OS-level push notifications.

**Reference:** PRD FR-PIN-006, §10.3

---

## Definition of Done

- [ ] `PassNotificationService` checks all pin pass times every 60 seconds
- [ ] When a pass starts within 5 minutes (configurable), a non-blocking in-app banner appears: "Approaching [Pin Name] — passes in X minutes"
- [ ] Banner shows the pin icon and name
- [ ] Tapping the banner navigates to the Pin Detail screen for that pin
- [ ] Each pin triggers at most one notification per pass (does not re-notify on every 60-second check)
- [ ] Banner auto-dismisses after 30 seconds
- [ ] Notification service starts at app launch and runs while the app is in the foreground

---

## Sub-tasks

- [ ] Create `lib/pins/pass_notification_service.dart`
- [ ] `PassNotificationService.start()`: start `Timer.periodic(Duration(minutes: 1), _check)`
- [ ] `_check()`: get current pin list and their pass times from `passTimesProvider`; for each pin, if `passStartUtc - DateTime.now() <= leadTime && passStartUtc > DateTime.now()` and not already notified for this pass (track `notifiedPasses` Set of `{pinId}_{passStartUtc}`): show notification
- [ ] Lead time stored in shared preferences key `pass_notification_lead_minutes`; default 5
- [ ] Notification UI: use `OverlayEntry` or `ScaffoldMessenger.showSnackBar` with a custom content widget showing pin icon, name, and countdown
- [ ] Navigation on tap: `Navigator.push(PinDetailScreen(pinId: pin.id))`
- [ ] Auto-dismiss: use `SnackBar(duration: Duration(seconds: 30))`
- [ ] `notifiedPasses` cleared on app restart (in-memory only; that is acceptable)
- [ ] Start `PassNotificationService` in `main.dart` after app is initialized
