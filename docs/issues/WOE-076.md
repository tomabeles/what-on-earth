# WOE-076: Telemetry HUD -- Magnetometer Interference Banner

**ID:** WOE-076
**Phase:** UI -- Telemetry HUD
**Depends on:** WOE-070, WOE-021
**UI Spec:** SS5.2

---

## Description

Add the magnetometer interference warning banner to the HUD. Displayed between the heading tape and pitch ladder when `DeviceOrientation.reliable == false`. Tapping the banner navigates to the Calibration screen.

Depends on WOE-021 (interference detection) for the `reliable` flag. Can be built with a mock `reliable` state and wired later.

---

## Definition of Done

- [ ] Banner renders between heading tape and pitch ladder when `reliable == false`
- [ ] Full-width, `hudBackground` backing
- [ ] Text: "!! MAG INTERFERENCE -- TAP TO RECALIBRATE" in `hudDanger` color
- [ ] Tapping the banner navigates to the Calibration screen (WOE-084)
- [ ] Banner dismissed automatically when `reliable` returns `true`
- [ ] Accepts `bool reliable` parameter
- [ ] Test: banner visible when reliable=false, hidden when reliable=true

---

## Sub-tasks

- [ ] This banner is better implemented as a Flutter widget overlay (not CustomPaint) since it needs tap handling
- [ ] Create `lib/shared/mag_interference_banner.dart`
- [ ] Widget: `Visibility` or `AnimatedOpacity` controlled by `reliable` state
- [ ] Layout: `Positioned` in the AR Stack, below heading tape area (~70dp from top), full-width with 16dp horizontal margin
- [ ] Container: `hudBackground` color, border radius 8dp, padding 8dp
- [ ] Row: warning icon + text, centered
- [ ] Text style: `hudFontFamily`, `hudFontSize`, `hudDanger` color
- [ ] `GestureDetector.onTap`: `Navigator.push(CalibrationScreen)` (or no-op until WOE-084 exists)
- [ ] Accept `bool reliable` from a provider (or mock for testing)
- [ ] Add to AR screen Stack, positioned between HUD heading tape and pitch ladder areas
- [ ] Widget test: render with reliable=false, verify banner visible; reliable=true, verify hidden
