# WOE-029: CesiumJS Base Raster Tile Imagery Layer Configuration

**ID:** WOE-029
**Phase:** 4 — Map Layers
**Step:** 4.5
**Depends on:** WOE-028, WOE-004

---

## Description

Configure a CesiumJS `UrlTemplateImageryProvider` pointing at the local tile server (`http://localhost:8765/tiles/base/{z}/{x}/{y}.webp`) as the globe's base imagery layer. At this stage, no tiles are downloaded yet — the layer will render blank areas wherever tiles are missing (404 responses from the tile server return no tile). That is expected and correct behavior until the tile downloader (WOE-030) populates the cache.

**Reference:** TECH_SPEC §3.5, §4.5, §7.3

---

## Definition of Done

- [ ] CesiumJS base imagery layer is configured using `UrlTemplateImageryProvider` pointed at `http://localhost:{TILE_SERVER_PORT}/tiles/base/{z}/{x}/{y}.webp`
- [ ] `minimumLevel: 0` and `maximumLevel: 5` are set (orbital-appropriate zoom range)
- [ ] Missing tiles (404) result in transparent/empty globe areas without CesiumJS errors in the console
- [ ] The layer is registered in `layersRegistry['base']` and toggleable via WOE-027
- [ ] When tiles are manually placed in the documents directory, they appear on the globe

---

## Sub-tasks

- [ ] In `web_globe/src/layers.js`, add `loadRasterLayers(viewer, tileServerPort)` function:
  ```javascript
  const baseProvider = new Cesium.UrlTemplateImageryProvider({
    url: `http://localhost:${tileServerPort}/tiles/base/{z}/{x}/{y}.webp`,
    minimumLevel: 0,
    maximumLevel: 5,
    tilingScheme: new Cesium.WebMercatorTilingScheme(),
    credit: 'OpenStreetMap contributors',
  });
  const baseLayer = viewer.imageryLayers.addImageryProvider(baseProvider);
  layersRegistry['base'] = baseLayer;
  ```
- [ ] Pass `TILE_SERVER_PORT` from Flutter to CesiumJS at init time via `SET_MODE` or a new `INIT_CONFIG` bridge message
- [ ] Add an `INIT_CONFIG` bridge message (Flutter → JS) sent just before `loadRasterLayers` is called, carrying `{ tileServerPort, cesiumLocalhostPort }`
- [ ] Handle 404 gracefully: CesiumJS does this by default for `UrlTemplateImageryProvider` (shows a transparent/missing tile indicator); confirm no JS errors thrown
- [ ] Similarly configure `layersRegistry['relief']` pointing at `/tiles/relief/{z}/{x}/{y}.webp`
- [ ] Add a couple of pre-downloaded test tiles manually to the documents directory via `adb push` (Android) and verify they appear on the globe
