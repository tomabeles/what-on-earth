# WOE-013: PositionController Riverpod Provider with Live/TLE Fallback

**ID:** WOE-013
**Phase:** 2 — Position Module
**Step:** 2.6
**Depends on:** WOE-009, WOE-012

---

## Description

Implement the `PositionController` Riverpod `AsyncNotifier` that manages the active position source and exposes a single unified `positionStream` to the rest of the app. It starts with `ISSLiveSource` as the active source. When the live source starts emitting `estimated` positions (indicating API failure), it switches to `TLESource`. When the live API recovers (first `live` position after an `estimated` run), it switches back.

Also implement the `PositionSourceStatus` model that the status indicator widget (WOE-015) will display.

**Reference:** TECH_SPEC §7.1

---

## Definition of Done

- [ ] `PositionController` is a Riverpod `AsyncNotifier` or `StateNotifier`
- [ ] On initialization, starts `ISSLiveSource`
- [ ] When 3 consecutive `estimated` positions are received from `ISSLiveSource`, switches to `TLESource`
- [ ] When a `live` position is received after being in TLE fallback, switches back to `ISSLiveSource`
- [ ] Exposes `Stream<OrbitalPosition> positionStream` that always emits from whichever source is currently active
- [ ] Exposes `PositionSourceStatus` (current source type, age of last fix, is-live boolean) as a reactive state
- [ ] Selected source mode is configurable via `setSourceMode(PositionSourceType)` for the settings screen
- [ ] Unit tests for fallback transition logic

---

## Sub-tasks

- [ ] Create `lib/position/position_controller.dart`
- [ ] Define `class PositionSourceStatus` with fields: `sourceType`, `lastFixAt` (DateTime?), `isLive` (bool)
- [ ] Implement `PositionController extends AsyncNotifier`:
  - `build()`: initialize `ISSLiveSource`, call `start()`, return initial `PositionSourceStatus`
  - Subscribe to active source's `positionStream`; track consecutive `estimated` count
  - On 3+ consecutive `estimated`: call `ISSLiveSource.stop()`, start `TLESource`, reset counter
  - On `live` position while TLE is active: stop `TLESource`, restart `ISSLiveSource`
  - Merge streams into a single broadcast `StreamController`
- [ ] Expose `Stream<OrbitalPosition> get positionStream` from the controller (or use a separate `positionStreamProvider`)
- [ ] Create `positionControllerProvider` using `riverpod_annotation` (`@riverpod`)
- [ ] Write unit tests with mocked position sources
