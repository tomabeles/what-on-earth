# WOE-089: Extract TouchSteeringController From ARScreen + AR Screen Widget Test

**ID:** WOE-089
**Phase:** Quality -- Refactoring
**Depends on:** WOE-087 (optional but recommended ordering)

---

## Description

`_ARScreenState` (`lib/screens/ar_screen.dart`) manages position/orientation/horizon subscriptions, camera + horizon detector lifecycle, orientation-lock and debug-toggle listeners, **and** the touch-steering state machine (override heading/pitch while dragging, 15%-per-tick ease-back on release, 0/360 wrap handling). The steering logic is pure math embedded in a widget and the app's main screen has no widget test.

Extract touch steering into a `TouchSteeringController` with a plain-Dart API (`onPanStart/Update/End`, `blend(orientation) → DeviceOrientation`, `isOverriding`), unit-test the wrap/ease-back math, and add a first `ar_screen_test.dart` widget test.

---

## Definition of Done

- [ ] `TouchSteeringController` in `lib/screens/touch_steering_controller.dart` (or `lib/sensors/`): no Flutter widget dependencies beyond gesture detail types
- [ ] `_ARScreenState` delegates pan callbacks and orientation blending to it; steering fields removed from the state class
- [ ] Unit tests for the controller: drag sensitivity, pitch clamp (0–180), heading wrap at 0/360, ease-back convergence and interruption by a new drag
- [ ] `test/screens/ar_screen_test.dart`: ARScreen pumps with mocked providers (fused orientation stream, position controller), asserts HUD receives updates and the steering gesture layer exists — camera/WebView stubbed or absent gracefully (existing code already catches camera init failure)
- [ ] No behavior change in manual drag/release on device

---

## Sub-tasks

- [ ] Define controller API; move `_isTouchSteering/_isReturning/_touchHeadingDeg/_touchPitchDeg` and the decay math into it
- [ ] Keep bridge sends in the screen (controller returns the blended orientation; screen sends)
- [ ] Write controller unit tests (pure, fast)
- [ ] Write the ARScreen widget test with provider overrides
- [ ] Run full suite + manual device check of touch steering feel
