# WOE-051: Battery Power-Saving Mode

**ID:** WOE-051
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.2
**Depends on:** WOE-017, WOE-019

---

## Description

Implement power-saving mode that activates automatically when device battery drops below 20%, reducing sensor fusion rate to 20 Hz and CesiumJS frame rate to 20 fps. This satisfies PRD NFR-BAT-002 and NFR-BAT-003.

**Reference:** PRD §7.2 NFR-BAT-001 through NFR-BAT-003

---

## Definition of Done

- [ ] `battery_plus` package added to `pubspec.yaml`
- [ ] When battery level drops below 20% (configurable threshold), power-saving mode activates:
  - Sensor fusion isolate reduces sample rate to `SensorInterval.ui` (~17 Hz, closest to 20 Hz available)
  - A `SET_FRAMERATE` bridge message is sent to CesiumJS: `viewer.targetFrameRate = 20`
- [ ] When battery level recovers above 30% (hysteresis), normal mode is restored
- [ ] A power-saving mode indicator is shown in the status bar when active (battery icon + "Power saver" label)
- [ ] Power-saving threshold is configurable in Settings (default 20%)

---

## Sub-tasks

- [ ] Add `battery_plus: ^6.0.2` to `pubspec.yaml`
- [ ] Create `lib/shared/power_mode_controller.dart` that watches `Battery.onBatteryStateChanged` and `Battery.batteryLevel`:
  - Poll battery level every 60 seconds (or on state change event)
  - If level < threshold and not already in power-save mode: activate
  - If level > (threshold + 10) and in power-save mode: deactivate
- [ ] `SensorFusionEngine.setSampleRate(SensorInterval rate)`: send a command message to the isolate to update `samplingPeriod` on all three sensor streams
- [ ] Add `SET_FRAMERATE` to the bridge `OutboundMessage` enum; in CesiumJS handle: `viewer.targetFrameRate = payload.fps`
- [ ] Create `powerModeProvider` Riverpod provider exposing a `bool isPowerSaving`
- [ ] Update `StatusBar` (WOE-015) to show a battery icon when `isPowerSaving` is true
- [ ] Add a "Power saver threshold" setting in `SettingsScreen` (slider, 10–50%)
