# WOE-033: Relief and Cloud Cover Imagery Layers

**ID:** WOE-033
**Phase:** 4 — Map Layers
**Step:** 4.9
**Depends on:** WOE-029, WOE-031

---

## Description

Configure the topographic relief shading and cloud cover raster imagery layers in CesiumJS. Relief tiles are served from the local tile server (offline). Cloud cover tiles are fetched from an online provider when connected and cached for up to 6 hours; the layer is hidden when cache is stale or offline.

**Reference:** TECH_SPEC §3.5, PRD FR-GLO-003

---

## Definition of Done

- [ ] Relief shading layer renders when tiles are present in the local cache
- [ ] Cloud cover layer renders when online and tiles are available; uses cached tiles up to 6 hours old when offline; hides automatically when cache is older than 6 hours
- [ ] Cloud cover layer has a visual "stale data" indicator in the layer toggle panel showing the cache age
- [ ] Both layers are toggleable via `TOGGLE_LAYER` bridge message
- [ ] Relief and cloud layers are registered in `layersRegistry`
- [ ] No errors in console when cloud cover layer is unavailable

---

## Sub-tasks

- [ ] In `layers.js`, add relief layer configuration (mirrors base layer setup with `/tiles/relief/` path)
- [ ] For cloud cover: the tile source URL is configurable (operator-provided; default to a public provider URL stored in `INIT_CONFIG` bridge message payload)
- [ ] In Flutter `TileCacheManager`, add `getLayerCacheAge(layerId)` returning the age of the oldest tile in the layer (proxy for freshness)
- [ ] In `GlobeView`, after `GLOBE_READY`, send `TOGGLE_LAYER` for cloud cover with `visible: false` if cache age > 6 hours (read from `TileCacheManager`)
- [ ] In the layer toggle panel (WOE-027), display a cache age label next to the cloud cover toggle: "Updated 2h ago" or "Unavailable (no cache)"
- [ ] Cloud cover TTL: store 6-hour value as a constant `cloudCacheTtlHours = 6` in `TileCacheManager`
- [ ] Background sync daemon (already started in WOE-031) should prioritize cloud cover tiles (shortest TTL) when connectivity is available
