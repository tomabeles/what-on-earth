# WOE-075: Telemetry HUD -- FPS Counter

**ID:** WOE-075
**Phase:** UI -- Telemetry HUD
**Depends on:** WOE-070
**UI Spec:** SS5.2

---

## Description

Add an FPS counter to the top-right of the HUD. Displays the frame rate reported by CesiumJS via `FRAME_RATE` bridge messages (sent every 5 seconds). Color changes at performance thresholds.

---

## Definition of Done

- [ ] FPS counter renders at top-right of the HUD: `"FPS: XX"`
- [ ] Font: `hudFontFamily` at `hudFontSize - 2`
- [ ] Color: `hudSecondary` when >= 25 fps; `hudWarning` when 15-24 fps; `hudDanger` when < 15 fps
- [ ] Accepts `int? fps` parameter; shows `"FPS: --"` when null
- [ ] `FRAME_RATE` inbound bridge message handler updates the fps value
- [ ] Test: verify color thresholds at 30, 20, 10 fps values

---

## Sub-tasks

- [ ] Add `paintFpsCounter(Canvas, Size, int? fps)` to `HudPainter`
- [ ] Position: top-right corner, below safe area, right-aligned with 16dp margin
- [ ] Color logic: `fps == null || fps >= 25 ? hudSecondary : fps >= 15 ? hudWarning : hudDanger`
- [ ] Format: `"FPS: ${fps ?? '--'}"`
- [ ] Use `TextPainter` with appropriate style
- [ ] Create `fpsProvider` (simple `StateProvider<int?>`, default null)
- [ ] In `BridgeController.registerHandlers()`: register `FRAME_RATE` handler that updates `fpsProvider` with `payload['fps']`
- [ ] Verify `InboundMessage.frameRate` already exists in bridge.dart (from WOE-006); if not, add it
- [ ] Add `fps` to `HudPainter` constructor
- [ ] Test: render with fps=30 (hudSecondary), fps=20 (hudWarning), fps=10 (hudDanger)
