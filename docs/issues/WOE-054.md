# WOE-054: Accessibility Implementation

**ID:** WOE-054
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.5
**Depends on:** WOE-023, WOE-039, WOE-042, WOE-049

---

## Description

Implement accessibility requirements across all Flutter UI screens. The globe itself (rendered in CesiumJS) is inherently inaccessible to screen readers, but all Flutter chrome — status bar, layer toggles, pin list, pin detail, onboarding, settings — must be fully accessible.

**Reference:** PRD §7.5 NFR-ACC-001 through NFR-ACC-003

---

## Definition of Done

- [ ] All interactive Flutter widgets have `Semantics` labels (tested with VoiceOver on iOS and TalkBack on Android)
- [ ] All icon-only buttons have `semanticsLabel` or `Tooltip`
- [ ] Text colors in all Flutter UI elements meet WCAG 2.1 AA contrast ratio (4.5:1 for normal text, 3:1 for large text) against their backgrounds
- [ ] System font scaling tested at 150%: no text truncation, no layout overflow
- [ ] `StatusBar` readable by VoiceOver/TalkBack: announces source type and data age as a semantic label
- [ ] Pin list items are fully traversable by screen reader
- [ ] All `TextFormField` inputs have correct `labelText` and `semanticsLabel`
- [ ] Accessibility test run documented in `docs/accessibility_review.md`

---

## Sub-tasks

- [ ] Enable accessibility inspector in iOS Simulator: Settings → Accessibility → VoiceOver; walk through all screens and identify any unlabeled elements
- [ ] Enable TalkBack on Android emulator: Settings → Accessibility → TalkBack; walk through all screens
- [ ] For each icon-only `IconButton`: add `tooltip:` parameter or wrap in `Semantics(label: '...')`
- [ ] For `StatusBar`: wrap the entire widget in `Semantics(label: 'Position: ${sourceType}, updated ${age}', liveRegion: true)`
- [ ] Review contrast ratios: use Flutter's `ThemeData` to ensure text uses `Theme.of(context).colorScheme.onSurface` or specific high-contrast colors; use an online contrast checker for custom colors
- [ ] Test font scaling: wrap app in `MediaQuery` override in tests, set `textScaleFactor: 1.5`; run widget tests and verify no `RenderFlex overflow` errors
- [ ] For CesiumJS globe: add a `Semantics(label: 'Live 3D globe view of Earth', excludeSemantics: true)` wrapper around `GlobeView` to prevent the WebView from being traversed as raw HTML by screen readers
- [ ] Write `docs/accessibility_review.md` with findings per screen
