# WOE-044: PinSync Differential Sync Implementation

**ID:** WOE-044
**Phase:** 6 — Cloud Sync and Onboarding
**Step:** 6.2
**Depends on:** WOE-043, WOE-035

---

## Description

Implement `PinSync` — the differential sync logic that synchronizes local pin data with Supabase when connectivity is available. Sync runs opportunistically: on connectivity change from offline to online, and on app foreground. The strategy is last-`updated_at`-wins per pin, with deleted pins propagated as tombstones.

**Reference:** TECH_SPEC §7.4, §4.4

---

## Definition of Done

- [ ] `PinSync` class exists in `lib/pins/pin_sync.dart`
- [ ] `sync()` method: fetches all rows from Supabase where `updated_at > lastSyncTimestamp`; for each remote row, upserts locally if remote `updated_at` is newer; pushes local rows where `updated_at > lastSyncTimestamp` to Supabase
- [ ] `lastSyncTimestamp` stored in shared preferences as `pin_sync_last_at` (ISO 8601 string); updated after each successful sync
- [ ] Deleted pins (where `deleted_at` is set) are synced to Supabase and filtered out on pull
- [ ] Sync is triggered on: `connectivity_plus` state change to online, app lifecycle `resumed` event
- [ ] Sync does not run if user is not authenticated (requires WOE-045; guard with a null-auth check)
- [ ] Sync errors are logged but do not surface to the user
- [ ] Integration test using a test Supabase project (optional — unit test with mocked Supabase client is required)

---

## Sub-tasks

- [ ] Create `lib/pins/pin_sync.dart` with `PinSync` class
- [ ] Inject `SupabaseClient` and `PinRepository` as dependencies
- [ ] Implement `Future<void> sync()`:
  - Pull: `supabase.from('pins').select().gt('updated_at', lastSyncTimestamp)` → for each row: if `localPin == null || remote.updated_at > local.updated_at`: upsert locally (including tombstone rows)
  - Push: query local DB for rows where `updated_at > lastSyncTimestamp`; `supabase.from('pins').upsert(rows.map(toSupabaseRow).toList())`
  - Update `lastSyncTimestamp` to current time on success
- [ ] Map `Pin` ↔ Supabase row (snake_case column names): implement `toSupabaseRow()` and `fromSupabaseRow()`
- [ ] Create `PinSyncController` Riverpod provider that:
  - Listens to `connectivity_plus` stream; calls `sync()` on transition to online
  - Listens to `WidgetsBindingObserver` lifecycle; calls `sync()` on `resumed`
- [ ] Guard: if `supabase.auth.currentUser == null`, skip sync silently
- [ ] Unit test with mocked `SupabaseClient`
