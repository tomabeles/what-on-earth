# WOE-068: Controls Button + Layer Control Panel

**ID:** WOE-068
**Phase:** UI -- AR Chrome
**Depends on:** WOE-063
**UI Spec:** SS3.2, SS5.3

---

## Description

Implement the "Controls" pill button in the bottom-left of the AR View and the Layer Control Panel overlay it triggers. The panel slides up from the button and provides toggle switches for map layers and camera.

Layer toggles send `TOGGLE_LAYER` bridge messages (from WOE-027) when that bridge message is available. Until then, toggles update local state only.

---

## Definition of Done

- [ ] `ControlsButton` widget: rectangular pill "^ Controls" in bottom-left, always visible
- [ ] Tapping the button toggles the `LayerControlPanel` overlay open/closed
- [ ] `LayerControlPanel`: left-anchored overlay above the Controls button
- [ ] Width 220dp, `surfaceOverlay` background, corner radius 12dp, row height 44dp
- [ ] Toggle rows: Camera, Relief shading, Cloud cover, Country borders, Coastlines, Cities & labels, Rivers & lakes
- [ ] Camera row hidden when `isMapMode` parameter is true
- [ ] Toggle active color: `hudPrimary`; inactive: `borderPrimary`
- [ ] All toggle states (except Camera) persisted in SharedPreferences and restored on launch
- [ ] Camera resets to ON on each cold launch
- [ ] Widget test: panel opens on button tap, toggles update state

---

## Sub-tasks

- [ ] Create `lib/shared/controls_button.dart`
- [ ] Button: `GestureDetector` wrapping a `Container` with `surfaceOverlay` bg, border radius 8, padding 8h/6v; `Text('^ Controls', style: hudFontFamily)`
- [ ] Create `lib/shared/layer_control_panel.dart`
- [ ] Panel: `AnimatedSlide` or `AnimatedPositioned` overlay, positioned directly above controls button
- [ ] Each row: `Row` with `Text(layerName)` and `Switch` (or `CupertinoSwitch`); active track color `hudPrimary`
- [ ] Layer IDs: `camera`, `relief`, `clouds`, `borders`, `coastlines`, `cities`, `rivers`
- [ ] State: Riverpod provider or `ValueNotifier` map of `{layerId: bool}`
- [ ] Persistence: read/write SharedPreferences keys `layer_visible_{layerId}` on toggle; defaults: all ON except `rivers` (OFF)
- [ ] Camera default: always ON on cold launch (do not read from prefs)
- [ ] When a toggle changes, call bridge `TOGGLE_LAYER` if `BridgeController` is available (optional dependency)
- [ ] `isMapMode` parameter: when true, filter out the camera row from the panel
- [ ] Widget test: render panel, toggle "borders", verify state updates; verify camera row hidden in map mode
