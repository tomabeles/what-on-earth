# WOE-011: satellite.js SGP4 Propagator and Bridge Messages

**ID:** WOE-011
**Phase:** 2 — Position Module
**Step:** 2.4
**Depends on:** WOE-006, WOE-010

---

## Description

Implement the SGP4 orbital propagation module inside the CesiumJS WebView bundle using `satellite.js`. This module: receives a TLE string from Flutter via the `SET_TLE` bridge message, initializes the `satellite.js` `satrec` object, propagates the ISS position every 2 seconds when called, and sends `POSITION_UPDATE` messages back to Flutter. This is the fallback propagation path used when the live API is unavailable.

**Reference:** TECH_SPEC §4.3, §8.1, §8.2

---

## Definition of Done

- [ ] `web_globe/src/satellite_propagator.js` exports `initTLE(line1, line2)` and `propagateNow()` as specified in TECH_SPEC §4.3
- [ ] `SET_TLE` bridge message (Flutter → JS) calls `initTLE` and starts a 2-second propagation interval
- [ ] Every 2 seconds, the propagator calls `propagateNow()` and sends a `POSITION_UPDATE` message back to Flutter via `callHandler`
- [ ] `POSITION_UPDATE` message payload matches `OrbitalPosition.fromJson()` format: `{lat, lon, altKm, timestamp, source: 'estimated'}`
- [ ] Propagation interval is cleared when a new `SET_TLE` message arrives (no duplicate intervals)
- [ ] When no TLE is set, no `POSITION_UPDATE` messages are sent
- [ ] Flutter `bridge.dart` registers a `POSITION_UPDATE` handler and converts the payload to an `OrbitalPosition`

## Sub-tasks

- [ ] Create `web_globe/src/satellite_propagator.js` with `initTLE` and `propagateNow` as per TECH_SPEC §4.3 code listing
- [ ] Add ECI → geodetic conversion using `satellite.eciToGeodetic` and `satellite.gstime`
- [ ] In `main.js`: handle `SET_TLE` message: call `initTLE(payload.line1, payload.line2)`, clear any existing interval, start a `setInterval` calling `propagateNow()` every 2000 ms
- [ ] In the interval callback: call `window.flutter_inappwebview.callHandler('POSITION_UPDATE', { lat, lon, altKm, timestamp: Date.now(), source: 'estimated' })`
- [ ] In Flutter `bridge.dart`: register `POSITION_UPDATE` handler; parse payload into `OrbitalPosition`; add it to a `StreamController` that `TLESource` will listen to
- [ ] Expose a `Stream<OrbitalPosition> get propagatedPositions` from `BridgeController`
- [ ] Manual test: send a known ISS TLE to the WebView, verify the output position matches a reference propagation (e.g., compare to wheretheiss.at output for the same TLE epoch)
