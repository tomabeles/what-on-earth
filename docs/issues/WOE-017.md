# WOE-017: SensorFusionEngine in Background Isolate

**ID:** WOE-017
**Phase:** 3 — Sensor Fusion and AR Camera
**Step:** 3.1
**Depends on:** WOE-001

---

## Description

Implement the `SensorFusionEngine` that runs a complementary filter in a background Dart `Isolate`. The engine subscribes to raw magnetometer, accelerometer, and gyroscope data from `sensors_plus` at 50 Hz and outputs a stable heading/pitch/roll tuple at the same cadence.

The complementary filter algorithm is specified in TECH_SPEC §7.2. The filter coefficient α = 0.98. The isolate communicates with the main isolate via a `ReceivePort` / `SendPort` pair.

**Reference:** TECH_SPEC §7.2

---

## Definition of Done

- [ ] `SensorFusionEngine` exists in `lib/sensors/sensor_fusion.dart`
- [ ] Runs in a background `Isolate` (not the main UI isolate)
- [ ] Subscribes to all three sensor streams at `SensorInterval.game` (~50 Hz)
- [ ] Implements the complementary filter: α = 0.98, gyroscope integration for short-term, accelerometer + magnetometer for drift correction
- [ ] Outputs heading (0–360°), pitch (−90 to +90°), roll (−180 to +180°) as a stream
- [ ] No calibration correction applied yet (that is WOE-020); raw magnetometer heading used
- [ ] Engine can be started and stopped cleanly
- [ ] Unit test for the filter math: given a sequence of synthetic sensor readings, verify the filter converges to the correct orientation

---

## Sub-tasks

- [ ] Create `lib/sensors/sensor_fusion.dart`
- [ ] Define `class RawSensorSample { DateTime ts; double x, y, z; }` for each sensor type
- [ ] Define `class OrientationSample { double headingDeg, pitchDeg, rollDeg; DateTime ts; bool reliable; }`
- [ ] Implement `SensorFusionEngine`:
  - `start()`: spawns an `Isolate` with `Isolate.spawn(_isolateEntryPoint, sendPort)`
  - The isolate entry subscribes to `sensors_plus` streams (magnetometer, accelerometer, gyroscope) at `SensorInterval.game`
  - Collects samples; on each gyroscope update, integrates gyro (dt × rad/s → angle delta); blends with accelerometer-derived pitch/roll and magnetometer-derived yaw using α = 0.98
  - Sends `OrientationSample` to main isolate via `SendPort`
  - The main isolate's `ReceivePort` listener publishes to a `StreamController<OrientationSample>`
- [ ] Implement the filter maths in a pure function `OrientationSample applyFilter(OrientationSample prev, RawSensorSample accel, RawSensorSample mag, RawSensorSample gyro, double dt)` for testability
- [ ] Write unit test for `applyFilter` with synthetic inputs
- [ ] Add `NSMotionUsageDescription` to `ios/Runner/Info.plist`
- [ ] Add `android.permission.BODY_SENSORS` (not required for IMU on Android, but verify in `AndroidManifest.xml`)
