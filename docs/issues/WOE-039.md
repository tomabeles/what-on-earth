# WOE-039: Pin List Screen

**ID:** WOE-039
**Phase:** 5 — Pins
**Step:** 5.6
**Depends on:** WOE-035, WOE-040

---

## Description

Implement the Pin List screen that displays all pins with a countdown to their next overhead pass. Tapping a pin in the list highlights it on the globe (in the 2D map view). This screen is navigable from the AR view.

Note: this issue depends on WOE-040 (Pass Calculator) for the pass countdown data, but the screen can be built first with a placeholder countdown. Reorder if needed.

**Reference:** TECH_SPEC §10.2, PRD FR-PIN-001 through FR-PIN-006

---

## Definition of Done

- [ ] `PinListScreen` exists in `lib/screens/pin_list_screen.dart`
- [ ] Shows all non-deleted pins as a scrollable list
- [ ] Each list tile shows: pin icon, name, formatted coordinates, and next pass countdown ("Next pass in X h Y m" or "Passing now!")
- [ ] List is sorted by next pass time (soonest first)
- [ ] Tapping a pin navigates to `PinDetailScreen` (WOE-042)
- [ ] Long-pressing a pin shows a context menu: "Edit", "Delete"
- [ ] Empty state: "No pins yet — tap the globe to add one"
- [ ] "+" FAB navigates back to the AR view to add a pin
- [ ] Widget tests for list rendering and empty state

---

## Sub-tasks

- [ ] Create `lib/screens/pin_list_screen.dart`
- [ ] Use `ref.watch(watchAllPinsProvider)` (wraps `PinRepository.watchAllPins()`) to get reactive pin list
- [ ] For each pin, look up next pass time from a `passTimesProvider` (populated by WOE-040); show placeholder "Calculating..." until available
- [ ] Use `ListView.builder` with `ListTile`: leading = icon widget, title = pin name, subtitle = `${lat}°, ${lon}°`, trailing = pass countdown
- [ ] Implement countdown formatting: `Duration` to "Xh Ym" or "< 1 min" or "Passing now"
- [ ] Sort: sort pin list by `nextPassTime` ascending before rendering
- [ ] Navigation: `Navigator.push(PinDetailScreen(pinId: pin.id))`
- [ ] Long-press context menu: use `showMenu` or `PopupMenuButton`; "Delete" calls `PinRepository.deletePin(pin.id)` with a confirmation `AlertDialog`
- [ ] Empty state: `Center(child: Column([Icon(Icons.place_outlined), Text('No pins yet...')]))` shown when list is empty
- [ ] Widget test: renders correct number of tiles, shows empty state when list is empty
