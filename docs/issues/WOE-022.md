# WOE-022: Camera Plugin Integration and Full-Screen Preview

**ID:** WOE-022
**Phase:** 3 — Sensor Fusion and AR Camera
**Step:** 3.6
**Depends on:** WOE-001

---

## Description

Integrate the Flutter `camera` plugin to display a live rear camera preview that fills the entire screen. This is the background layer of the AR view — it shows the physical world through the device's viewport. The camera is set to medium resolution to balance visual quality with performance and battery impact.

**Reference:** TECH_SPEC §3.1, PRD NFR-BAT-001

---

## Definition of Done

- [ ] `CameraController` is initialized with the rear camera at `ResolutionPreset.medium`
- [ ] `CameraPreview` widget fills the full screen (no letterboxing)
- [ ] Camera permission is requested at runtime (iOS: `NSCameraUsageDescription` in `Info.plist`; Android: `CAMERA` permission in `AndroidManifest.xml`)
- [ ] If camera permission is denied, a graceful fallback screen is shown ("Camera permission required for AR view")
- [ ] `CameraController` is properly disposed when the widget is removed from the tree
- [ ] `flutter analyze` passes on new code

---

## Sub-tasks

- [ ] Add `NSCameraUsageDescription` to `ios/Runner/Info.plist` with a user-facing reason
- [ ] Add `<uses-permission android:name="android.permission.CAMERA" />` to `android/app/src/main/AndroidManifest.xml`
- [ ] Create `lib/globe/camera_preview_layer.dart` widget:
  - `initState`: enumerate cameras, select first rear camera, create `CameraController(camera, ResolutionPreset.medium)`, call `controller.initialize()`
  - Return `CameraPreview(controller)` wrapped in a `SizedBox.expand()` or `LayoutBuilder` to force full-screen fill
  - Handle initialization errors
  - `dispose()`: `controller.dispose()`
- [ ] Handle permission denied: use `permission_handler` package or handle `CameraException` with code `cameraAccessDenied`; show a `Scaffold` with an explanatory message and a "Open Settings" button
- [ ] Test on both iOS and Android physical devices
