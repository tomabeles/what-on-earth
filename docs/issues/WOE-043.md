# WOE-043: Supabase Project Setup and Pins Table Schema

**ID:** WOE-043
**Phase:** 6 — Cloud Sync and Onboarding
**Step:** 6.1
**Depends on:** WOE-034

---

## Description

Create the Supabase project and define the `pins` table with Row Level Security (RLS) policies that restrict each user to their own data. This is the cloud backend for pin sync.

**Reference:** TECH_SPEC §4.4, §9.3

---

## Definition of Done

- [ ] Supabase project created (can be a development project; production project setup deferred to deployment)
- [ ] `pins` table created with schema matching TECH_SPEC §9.3: `id` (uuid PK), `user_id` (uuid FK to auth.users), `lat_deg` (float8), `lon_deg` (float8), `name` (text), `note` (text nullable), `icon_id` (int2 default 0), `created_at` (timestamptz), `updated_at` (timestamptz), `deleted_at` (timestamptz nullable)
- [ ] RLS enabled on `pins` table
- [ ] RLS policies: `SELECT` where `auth.uid() = user_id`; `INSERT` where `auth.uid() = user_id`; `UPDATE` where `auth.uid() = user_id`; `DELETE` blocked (soft deletes only — `UPDATE` policy covers setting `deleted_at`)
- [ ] A SQL migration file is committed to `supabase/migrations/` in the repo
- [ ] `SUPABASE_URL` and `SUPABASE_ANON_KEY` are documented in `.env.example`

---

## Sub-tasks

- [ ] Create a new Supabase project at `supabase.com` (or self-host for dev)
- [ ] In the Supabase SQL editor, run the migration:
  ```sql
  CREATE TABLE pins (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    lat_deg float8 NOT NULL,
    lon_deg float8 NOT NULL,
    name text NOT NULL CHECK (char_length(name) <= 100),
    note text,
    icon_id smallint NOT NULL DEFAULT 0,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now(),
    deleted_at timestamptz
  );
  ALTER TABLE pins ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "Users can manage their own pins" ON pins
    USING (auth.uid() = user_id)
    WITH CHECK (auth.uid() = user_id);
  CREATE INDEX pins_user_updated ON pins(user_id, updated_at DESC);
  ```
- [ ] Save the SQL as `supabase/migrations/001_create_pins.sql`
- [ ] Create `supabase/` directory at repo root
- [ ] Copy `SUPABASE_URL` and `SUPABASE_ANON_KEY` (anon key, not service key) from Supabase dashboard into `.env.example`
- [ ] Verify RLS by testing that a request with no auth header returns 0 rows
