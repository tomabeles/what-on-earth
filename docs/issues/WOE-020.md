# WOE-020: Magnetometer Calibration Store and Hard-Iron Correction

**ID:** WOE-020
**Phase:** 3 — Sensor Fusion and AR Camera
**Step:** 3.4
**Depends on:** WOE-017

---

## Description

Implement persistent storage and application of magnetometer hard-iron calibration parameters. Hard-iron bias is a constant offset in the magnetometer readings caused by permanent magnets in the device or nearby. It is the simplest and most impactful magnetometer correction.

The calibration parameters are stored in `flutter_secure_storage` using the schema from TECH_SPEC §9.5. When present, they are applied in the sensor fusion engine as a pre-processing step on each raw magnetometer sample before the complementary filter runs.

The calibration routine UI is implemented later in WOE-047. This issue implements the storage and application layer only.

**Reference:** TECH_SPEC §7.2, §9.5

---

## Definition of Done

- [ ] `CalibrationStore` class exists in `lib/sensors/calibration.dart`
- [ ] `save(CalibrationParams params)` writes JSON to `flutter_secure_storage` key `mag_calibration`
- [ ] `load()` reads and deserializes; returns null if not present
- [ ] `SensorFusionEngine` reads calibration on start and applies hard-iron offset to raw magnetometer readings
- [ ] When `save()` is called with new params (e.g., after recalibration), the engine updates its correction in real time (no restart required)
- [ ] Unit tests: save → load round-trip; verify correction is applied to synthetic magnetometer readings

## Sub-tasks

- [ ] Create `lib/sensors/calibration.dart`:
  - `class CalibrationParams` with `hardIron` (List<double> length 3), `softIron` (List<List<double>> 3×3), `calibratedAt` (DateTime), `confidence` (double 0–1)
  - `class CalibrationStore` with `save(CalibrationParams)` and `Future<CalibrationParams?> load()`
  - Use `flutter_secure_storage` key `'mag_calibration'`; serialize/deserialize as JSON
- [ ] Update `SensorFusionEngine` isolate entry point:
  - At startup, call `CalibrationStore.load()` and store params
  - Before the complementary filter, subtract hard-iron bias: `correctedX = rawX - hardIron[0]` (and y, z)
  - Expose a `SendPort` command to update calibration params without restarting the isolate
- [ ] Add a `updateCalibration(CalibrationParams)` method to `SensorFusionEngine`
- [ ] Unit test: apply a known hard-iron offset and verify corrected readings are as expected
