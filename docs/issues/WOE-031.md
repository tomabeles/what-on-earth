# WOE-031: TileCacheManager with LRU Eviction

**ID:** WOE-031
**Phase:** 4 — Map Layers
**Step:** 4.7
**Depends on:** WOE-030

---

## Description

Implement the `TileCacheManager` that tracks tile metadata in a drift database, enforces a maximum cache size via LRU eviction, and provides cache status information (total size, last sync time, tile counts) to the UI.

**Reference:** TECH_SPEC §7.3, §9.4

---

## Definition of Done

- [ ] `TileCacheManager` exists in `lib/tile_cache/cache_manager.dart`
- [ ] `TileMetadata` drift table schema matches TECH_SPEC §9.4 exactly
- [ ] On each tile download (called from `TileDownloader`), metadata is inserted or updated
- [ ] On each tile serve (called from tile server when a tile is accessed), `lastAccessedAt` is updated
- [ ] `enforceMaxSize(int maxBytes)` evicts LRU tiles (ordered by `lastAccessedAt ASC`) until total size is under the limit; evicted tiles are deleted from disk and from metadata DB
- [ ] Default max size: 3 GB (3 × 1024 × 1024 × 1024 bytes)
- [ ] `CacheStatus getCacheStatus()` returns `{totalSizeBytes, tileCount, lastSyncAt, layerStatuses}`
- [ ] Unit tests for LRU eviction logic

---

## Sub-tasks

- [ ] Generate drift code for `TileMetadata` table (add to existing `pin_database.dart` or create a separate `tile_database.dart`)
- [ ] Create `lib/tile_cache/cache_manager.dart` with `TileCacheManager`
- [ ] Implement `recordDownload(TileCoordinate, int fileSizeBytes)`: insert or replace row in `tile_metadata` table
- [ ] Implement `recordAccess(TileCoordinate)`: update `lastAccessedAt` for the tile; call this from the `shelf` tile server handler when a tile is served
- [ ] Implement `enforceMaxSize(int maxBytes)`: query total `SUM(fileSizeBytes)` from DB; if over limit, query LRU tiles; delete each file and then delete DB row; repeat until under limit
- [ ] Implement `CacheStatus getCacheStatus()`: aggregate query from DB
- [ ] Call `enforceMaxSize` after each `TileDownloader.downloadLayer()` completes
- [ ] Wire metadata recording into `TileDownloader`: accept `TileCacheManager` as dependency, call `recordDownload` after each successful tile write
- [ ] Run `build_runner` to regenerate drift code
- [ ] Unit tests in `test/unit/cache_manager_test.dart`
