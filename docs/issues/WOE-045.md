# WOE-045: Supabase OAuth 2.0 PKCE Auth Flow

**ID:** WOE-045
**Phase:** 6 — Cloud Sync and Onboarding
**Step:** 6.3
**Depends on:** WOE-043

---

## Description

Implement Supabase authentication using OAuth 2.0 with PKCE via `supabase_flutter`. Authentication is optional — the app works fully without it (pins are local only). When the user signs in, cloud sync is enabled.

Per PRD NFR-SEC-001 and the PRD's note on configurable identity providers, the auth provider is configurable (the Supabase project can be configured with any OAuth provider: Google, GitHub, Apple, etc.).

**Reference:** TECH_SPEC §4.4, PRD NFR-SEC-001

---

## Definition of Done

- [ ] `Supabase.initialize(url, anonKey)` is called in `main.dart` before `runApp`
- [ ] A `SignInScreen` exists that presents an OAuth sign-in button (e.g., "Sign in with Google")
- [ ] On successful sign-in, the user session is persisted by `supabase_flutter` (no re-auth on app restart)
- [ ] On sign-out, session is cleared and cloud sync stops
- [ ] Auth state is exposed as a Riverpod `StreamProvider<AuthState>`
- [ ] The Settings screen shows the signed-in user's email and a "Sign Out" button, or a "Sign In" button if not authenticated
- [ ] Sign-in is not required during onboarding (user can skip)

---

## Sub-tasks

- [ ] In `main.dart`, add:
  ```dart
  await Supabase.initialize(
    url: const String.fromEnvironment('SUPABASE_URL'),
    anonKey: const String.fromEnvironment('SUPABASE_ANON_KEY'),
  );
  ```
- [ ] Configure deep link redirect URL in Supabase dashboard for the OAuth callback: `com.whatonearth://login-callback/`
- [ ] Add deep link scheme to `ios/Runner/Info.plist` (`CFBundleURLTypes`) and `android/app/src/main/AndroidManifest.xml` (intent filter)
- [ ] Create `lib/screens/sign_in_screen.dart`:
  - Button: "Sign in with [provider]"
  - On tap: `supabase.auth.signInWithOAuth(OAuthProvider.google, redirectTo: 'com.whatonearth://login-callback/')`
  - Handle deep link callback using `app_links` or `supabase_flutter`'s built-in deep link handling
- [ ] Create `authStateProvider` as `StreamProvider<AuthState>` wrapping `supabase.auth.onAuthStateChange`
- [ ] Update Settings screen (WOE-049) to show auth state and sign-out button
- [ ] Sign-in screen accessible from Settings ("Connect for cloud sync")
