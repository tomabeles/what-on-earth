# WOE-057: USB Tile Pre-seed Tooling

**ID:** WOE-057
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.8
**Depends on:** WOE-028, WOE-030

---

## Description

Implement and document `scripts/seed_tiles.sh`, a command-line tool that downloads a full tile cache to a local directory on a development machine. The directory can then be transferred to a device via USB (`adb push` on Android, iTunes File Sharing on iOS) for pre-seeding the tile cache before a mission when no over-the-air download is possible.

**Reference:** TECH_SPEC §11.2, PRD NFR-CMP-003

---

## Definition of Done

- [ ] `scripts/seed_tiles.sh` downloads all tiles for all layers at zoom 0–5 to a local `tile_cache/` directory
- [ ] Output directory structure matches the device's tile storage layout: `tile_cache/tiles/{layer}/{z}/{x}/{y}.ext`
- [ ] Script is idempotent: skips tiles that already exist locally
- [ ] Script accepts arguments: `--layers base,relief,clouds`, `--zoom 0-5`
- [ ] `docs/usb_tile_seed.md` documents the full transfer procedure for Android and iOS
- [ ] Android transfer tested via `adb push`
- [ ] iOS transfer approach documented (iTunes File Sharing or `ideviceinstaller`)

---

## Sub-tasks

- [ ] Create `scripts/seed_tiles.sh`:
  ```bash
  #!/usr/bin/env bash
  # Usage: ./seed_tiles.sh --layers base,relief --zoom 0-5
  LAYERS="${LAYERS:-base,relief,clouds}"
  ZOOM_MAX="${ZOOM_MAX:-5}"
  OUTPUT_DIR="${OUTPUT_DIR:-./tile_cache}"
  # Enumerate tiles, download with curl, save to OUTPUT_DIR/tiles/{layer}/{z}/{x}/{y}.ext
  ```
  - Use `curl` with `-z` (skip if not modified) for idempotent downloads
  - Tile URL templates: configurable via environment variables (same as `TileDownloader` source URLs)
  - Concurrent downloads: `xargs -P 4`
- [ ] Create `docs/usb_tile_seed.md`:
  - **Android:** `adb push tile_cache/tiles /sdcard/Android/data/com.whatonearth/files/tiles`; verify path matches `path_provider.getApplicationDocumentsDirectory()` on Android
  - **iOS:** Enable iTunes File Sharing in `Info.plist` (`UIFileSharingEnabled: YES`); in Finder (macOS) or iTunes: select device → Files → What On Earth → drag `tiles/` folder
  - Include a note about needing to restart the app after tile pre-seed so the tile server picks up the new files
- [ ] Verify tile file paths from `adb push` match what the `shelf` tile server expects
- [ ] Test end-to-end: seed tiles on desktop, push to Android device, launch app in airplane mode, verify tiles appear on globe
