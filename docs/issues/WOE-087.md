# WOE-087: Extract Pure Filter Core From SensorFusionEngine

**ID:** WOE-087
**Phase:** Quality -- Refactoring
**Depends on:** none (touches code from the SENSOR_FUSION_SPEC rewrite)

---

## Description

`SensorFusionEngine` (`lib/sensors/sensor_fusion.dart`, ~730 lines) mixes three concerns: (1) sensor stream subscriptions and lifecycle, (2) the complementary-filter math, and (3) debug toggles/diagnostics. Testing the filter requires driving fake sensor streams; the math cannot be exercised directly.

Extract the filter into a pure, stateless (or explicitly-state-passing) class — input: accel/gyro/mag samples + dt + mode/calibration; output: `DeviceOrientation` + diagnostics. `SensorFusionEngine` keeps subscriptions, lifecycle, and feeds the filter.

Also extract the ZXY Euler rotation-matrix construction that is currently duplicated in `debug_coord_axis_overlay.dart` and `debug_mag_ref_overlay.dart` into one shared, tested function — the orientation math must have a single home.

This is the highest-value testability refactor in the app: sensor fusion is safety-critical for the AR experience and currently only testable end-to-end.

- [ ] No behavioral change: same outputs for same inputs (verify by running the existing tests before and after)
