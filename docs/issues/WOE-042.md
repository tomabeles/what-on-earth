# WOE-042: Pin Detail Screen

**ID:** WOE-042
**Phase:** 5 — Pins
**Step:** 5.9
**Depends on:** WOE-035, WOE-040

---

## Description

Implement the Pin Detail screen that shows full information for a single pin: name, note, coordinates, icon, next pass schedule, and edit/delete actions.

**Reference:** TECH_SPEC §10.2, PRD FR-PIN-002, FR-PIN-003

---

## Definition of Done

- [ ] `PinDetailScreen` exists in `lib/screens/pin_detail_screen.dart`
- [ ] Accepts a `pinId` parameter
- [ ] Displays: pin icon (large), name, note (if present), formatted coordinates, next pass time (full datetime), max elevation, pass duration
- [ ] Pass information shows a live countdown: "Next pass in X h Y m Z s" updated every second
- [ ] "Edit" action opens an edit bottom sheet (same as `AddPinSheet` but pre-filled)
- [ ] "Delete" action shows a confirmation dialog then calls `PinRepository.deletePin` and pops the screen
- [ ] If the pass has started ("Passing now!"), a prominent indicator is shown
- [ ] If no pass is found within 48 hours, shows "No pass predicted in the next 48 hours"

---

## Sub-tasks

- [ ] Create `lib/screens/pin_detail_screen.dart`
- [ ] Load pin from `PinRepository.getPinById(pinId)` and watch for changes
- [ ] Load pass result from `passTimesProvider.forPin(pinId)`
- [ ] Live countdown: use a `Stream.periodic(Duration(seconds: 1))` or a `Ticker` to rebuild the countdown widget every second
- [ ] Edit action: `showModalBottomSheet` with `AddPinSheet` pre-filled with current pin data; `AddPinSheet` should support an optional existing `Pin` parameter and call `PinRepository.updatePin` instead of `createPin` when provided
- [ ] Delete action: `showDialog` with `AlertDialog(title: 'Delete pin?', actions: [Cancel, Delete])`; on confirm: `await ref.read(pinRepositoryProvider).deletePin(pin.id)` then `Navigator.pop(context)`
- [ ] "Passing now" indicator: show a pulsing green dot and "Passing now!" text when `DateTime.now()` is between `passStartUtc` and `passStartUtc + Duration(seconds: passDurationSeconds)`
- [ ] Widget test: countdown displays correctly; "passing now" indicator appears when in-window
