# WOE-065: Speed-Dial NAV FAB Component

**ID:** WOE-065
**Phase:** UI -- AR Chrome
**Depends on:** WOE-063
**UI Spec:** SS3.1, SS5.4

---

## Description

Implement the speed-dial floating action button that serves as the primary navigation control. Pinned to the bottom-right of the AR View. Closed state shows a grid icon; open state shows X with three secondary action FABs animating upward with label pills.

This ticket builds the reusable component only. Navigation wiring is in WOE-066.

---

## Definition of Done

- [ ] `NavSpeedDial` widget exists in `lib/shared/nav_speed_dial.dart`
- [ ] Closed: single 56dp circular FAB with grid icon, `fabBackground`/`fabIcon` colors
- [ ] Open: icon changes to X; three 40dp secondary FABs animate upward
- [ ] Secondary FABs: globe-crosshair (Map), list-pin (Pins), gear (Settings) icons
- [ ] Each secondary FAB has a label pill to its left (`surfaceOverlay` bg, `hudPrimary` text)
- [ ] Animation: scale 0->1 + fade in, staggered 60ms apart, upward from primary
- [ ] Tapping outside the expanded FAB collapses it
- [ ] `activeDestination` parameter highlights the matching secondary FAB with `hudPrimary` bg
- [ ] Callbacks: `onMapTap`, `onPinsTap`, `onSettingsTap`
- [ ] Widget test: FAB expands on tap, shows 3 secondary buttons, collapses on outside tap

---

## Sub-tasks

- [ ] Create `lib/shared/nav_speed_dial.dart`
- [ ] Primary FAB: `FloatingActionButton` 56dp with `AnimatedSwitcher` for grid/close icon transition
- [ ] State: `_isOpen` boolean controlling expansion
- [ ] Secondary FABs: 3x `FloatingActionButton.small` (40dp) in a `Column` above primary
- [ ] Animation: use `AnimatedScale` + `AnimatedOpacity` or `SlideTransition` with staggered `AnimationController` (total duration ~240ms for 3 items at 60ms stagger)
- [ ] Label pills: `Container` with `surfaceOverlay` background, `borderRadius: 8`, padding 8h/4v, text in `hudPrimary` at 12pt
- [ ] Gap between secondary FABs: 8dp
- [ ] Tap outside: wrap in `GestureDetector` or use a `ModalBarrier` behind the expanded FABs
- [ ] Active highlight: when `activeDestination` matches a button, set its background to `hudPrimary` and icon to `fabIcon`
- [ ] Read `AppTokens` via `Theme.of(context).extension<AppTokens>()!`
- [ ] Widget test: render, tap primary, verify 3 secondary FABs appear; tap outside, verify collapsed
