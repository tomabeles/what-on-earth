# WOE-069: SET_SKYBOX Bridge Message + Viewer Init

**ID:** WOE-069
**Phase:** UI -- Bridge
**Depends on:** WOE-006
**UI Spec:** SS6.1

---

## Description

Add the `SET_SKYBOX` outbound bridge message and its CesiumJS handler. Also modify the CesiumJS Viewer constructor to use the default skybox (star catalog cube map) instead of `skyBox: false`, then immediately hide it at startup. This enables runtime toggling of the star background when the camera is turned off.

No UI wiring in this ticket -- that is done in WOE-077 (Camera Toggle).

---

## Definition of Done

- [ ] `SET_SKYBOX` added to `OutboundMessage` enum in `bridge.dart` with `messageName` getter
- [ ] `BridgeController` has a `setSkybox(bool enabled)` convenience method
- [ ] CesiumJS `main.js` Viewer constructor: remove `skyBox: false`, keep `skyAtmosphere: false`
- [ ] After Viewer init: `viewer.scene.skyBox.show = false` (camera ON by default)
- [ ] JS handler `SET_SKYBOX` toggles `viewer.scene.skyBox.show` and `viewer.scene.backgroundColor` (opaque black when skybox on, transparent when off)
- [ ] Bridge test: verify `SET_SKYBOX` serialization
- [ ] Regression: globe still loads and renders correctly after Viewer init change

---

## Sub-tasks

- [ ] In `lib/globe/bridge.dart`: add `setSkybox` to `OutboundMessage` enum; add `.messageName` case
- [ ] Add `void setSkybox(bool enabled)` method to `BridgeController`: calls `send(OutboundMessage.setSkybox, {'enabled': enabled})`
- [ ] In `web_globe/src/main.js`: remove `skyBox: false` from Viewer constructor options
- [ ] Add `viewer.scene.skyBox.show = false;` after Viewer creation (before `GLOBE_READY`)
- [ ] Add handler:
  ```javascript
  handlers['SET_SKYBOX'] = ({ enabled }) => {
    viewer.scene.skyBox.show = enabled;
    viewer.scene.backgroundColor = enabled
      ? new Cesium.Color(0, 0, 0, 1)
      : new Cesium.Color(0, 0, 0, 0);
  };
  ```
- [ ] Run `scripts/build_globe.sh` to rebuild
- [ ] Bridge test: `OutboundMessage.setSkybox.messageName` equals `'SET_SKYBOX'`
- [ ] Manual test: globe still loads correctly; no visual regression
