# WOE-004: GlobeView Widget with Transparent WebView

**ID:** WOE-004

**Step:** 1.4
**Depends on:** WOE-002, WOE-003

## Description

Implement the `GlobeView` Flutter widget that hosts the CesiumJS globe inside a `flutter_inappwebview` `InAppWebView`. The widget starts `InAppLocalhostServer` on port 8080 to serve the `assets/globe/` bundle, loads `index.html`, and configures the WebView with a transparent background.

At this stage, the globe renders with a static camera position at ISS altitude (~420,000 m) looking straight down at 0┬░N, 0┬░E. No position data, no sensors ŌĆö just a visible globe.

The CesiumJS `main.js` must configure the transparent viewer (all settings from TECH_SPEC ┬¦3.2) and emit the `GLOBE_READY` message when initialized.

**Reference:** TECH_SPEC ┬¦3.2, ┬¦3.3, ┬¦6.1 (`lib/globe/globe_view.dart`)

---

## Definition of Done

- [ ] `GlobeView` widget renders without errors on iOS and Android
- [ ] CesiumJS globe is visible at ISS orbital altitude, looking down at Earth
- [ ] WebView background is transparent (no white or black fill behind the globe)
- [ ] `InAppLocalhostServer` starts on port 8080 and serves the globe bundle
- [ ] CesiumJS `GLOBE_READY` event fires and is received by Flutter (logged to console)
- [ ] Globe renders in dark mode (black space around Earth, not white)
- [ ] No Cesium Ion token errors in the browser console (imagery source is not configured yet; that is fine)

## Sub-tasks

- [ ] Add `flutter_inappwebview` initialization in `main.dart` (call `InAppWebViewController.setWebContentsDebuggingEnabled(true)` in debug mode)
- [ ] Create `lib/globe/globe_view.dart`:
  - Start `InAppLocalhostServer(port: 8080)` in `initState`
  - Stop it in `dispose`
  - Return an `InAppWebView` with `initialUrlRequest: URLRequest(url: WebUri('http://localhost:8080/index.html'))`
  - Set `InAppWebViewSettings(transparentBackground: true, useHybridComposition: false)` (Android fix)
- [ ] Update `web_globe/src/main.js` with full transparent viewer configuration from TECH_SPEC ┬¦3.2:
  - `contextOptions: { webgl: { alpha: true } }`
  - `skyBox: false`, `skyAtmosphere: false`
  - `viewer.scene.backgroundColor = new Cesium.Color(0, 0, 0, 0)`
  - `viewer.scene.highDynamicRange = false`
  - `viewer.scene.globe.translucency.enabled = true`
  - Disable all default UI widgets
- [ ] Set static camera position: `viewer.camera.setView({ destination: Cesium.Cartesian3.fromDegrees(0, 0, 420000000) })` (note: meters, not km; ISS at 420 km = 420,000 m; but wait ŌĆö 420 km = 420,000 m; altitude in Cesium is meters above ellipsoid)

  Actually: `Cesium.Cartesian3.fromDegrees(lon, lat, altMeters)` where altMeters for ISS = 420,000 meters = 420 km. The camera is placed at the ISS position and looks down. Set pitch to -90┬░ (straight down) initially.

- [ ] Emit `GLOBE_READY` via `window.flutter_inappwebview.callHandler('GLOBE_READY', {})` after viewer init
- [ ] Add `addJavaScriptHandler(handlerName: 'GLOBE_READY', callback: ...)` in Flutter to log receipt
- [ ] Wire `GlobeView` into `lib/app.dart` as the home screen
- [ ] Run on device and verify globe is visible
