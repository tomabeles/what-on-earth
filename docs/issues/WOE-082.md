# WOE-082: Settings -- Layers + Tile Cache Sections

**ID:** WOE-082
**Phase:** UI -- Settings
**Depends on:** WOE-068, WOE-031
**UI Spec:** SS4.6

---

## Description

Add the Layers and Tile Cache sections to the Settings screen. The Layers section mirrors the Control Panel toggles and syncs bidirectionally. The Tile Cache section shows cache status and management actions.

---

## Definition of Done

- [ ] **Layers section:** toggle switches for all layers (same list as Control Panel)
- [ ] Toggles sync bidirectionally with Control Panel state (shared provider)
- [ ] **Tile Cache section:**
  - Cache size used (formatted, e.g., "1.2 GB")
  - Last sync timestamp (relative, e.g., "3 days ago")
  - Per-layer coverage: green check / amber warning / red X per layer
  - "Re-download All" action button
  - "Clear Cache" destructive action with confirmation dialog
- [ ] Widget test: layer toggle in settings updates shared state; clear cache shows confirmation

---

## Sub-tasks

- [ ] Add Layers section card to Settings ListView
- [ ] Reuse the same layer state provider from WOE-068; each `SwitchListTile` watches and updates the shared state
- [ ] Changes in Settings propagate to Control Panel and vice versa (same Riverpod provider)
- [ ] Add Tile Cache section card to Settings ListView
- [ ] Cache size: read from `TileCacheManager.getCacheStatus()` (or show "Not available" until WOE-031 exists)
- [ ] Last sync: format `lastSyncAt` as relative time using `DateTime.now().difference(lastSync)`
- [ ] Per-layer coverage: `Row` of layer name + icon (check=complete, warning=partial, X=missing)
- [ ] "Re-download All": `ElevatedButton` that triggers tile download for all layers
- [ ] "Clear Cache": `TextButton.icon` with destructive red color; on tap shows `AlertDialog` with "This will delete all cached map tiles. You'll need to re-download for offline use." + Cancel/Clear actions
- [ ] On Clear confirm: call `TileCacheManager.clearAll()` and update displayed size
- [ ] Widget test: render, verify layer toggles present; verify clear cache dialog appears
