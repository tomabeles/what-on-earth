# WOE-027: TOGGLE_LAYER Bridge Message and Layer Controls UI

**ID:** WOE-027
**Phase:** 4 — Map Layers
**Step:** 4.3
**Depends on:** WOE-026, WOE-006

---

## Description

Implement the `TOGGLE_LAYER` Flutter→JS bridge message and the collapsible layer toggle panel in the AR view UI. Users can show/hide individual map layers from within the AR experience.

**Reference:** TECH_SPEC §8.1, §10.2

---

## Definition of Done

- [ ] `TOGGLE_LAYER` bridge message implemented: `{ layerId: string, visible: boolean }`
- [ ] CesiumJS `TOGGLE_LAYER` handler sets `layersRegistry[layerId].show = visible`
- [ ] A collapsible layer panel in the AR view shows toggle switches for: borders, coastlines, relief, cloud cover, cities
- [ ] Toggle state persists in shared preferences across app restarts
- [ ] Layer toggle controls are one-handed reachable (bottom-right corner of screen)
- [ ] Widget tests for the layer toggle panel

---

## Sub-tasks

- [ ] In CesiumJS `main.js`, implement `TOGGLE_LAYER` handler: look up `layersRegistry[payload.layerId]` and set `.show = payload.visible`; also handle `ImageryLayer` show toggle for raster layers
- [ ] In Flutter `bridge.dart`, add `toggleLayer(String layerId, bool visible)` convenience method
- [ ] Create `lib/shared/layer_toggle_panel.dart`: an `AnimatedContainer` that expands/collapses; contains `SwitchListTile` widgets for each layer
- [ ] Layer IDs: `'borders'`, `'coastlines'`, `'relief'`, `'clouds'`, `'cities'`
- [ ] Store toggle state in shared preferences: key `layer_visible_{layerId}`; default all to `true` except `clouds` (default `false` until tile cache is populated)
- [ ] On AR view init, send `TOGGLE_LAYER` for each layer with the stored state
- [ ] Widget test: toggling a switch calls `bridgeController.toggleLayer` with correct args
