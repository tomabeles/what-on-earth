# WOE-056: MDM Packaging and Distribution

**ID:** WOE-056
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.7
**Depends on:** WOE-007

---

## Description

Build and test enterprise distribution packages for both iOS and Android that can be deployed via MDM (Mobile Device Management) without the public App Store. Document the MDM configuration profile format so that agency IT teams can pre-configure the app before crew deployment.

**Reference:** TECH_SPEC §11.2, PRD §11.2 NFR-CMP-002

---

## Definition of Done

- [ ] iOS: `.ipa` built using Apple Enterprise Distribution certificate (or ad-hoc profile for testing); can be installed on a device without App Store
- [ ] Android: `.apk` built in release mode; installable via `adb install` without Google Play
- [ ] Both packages built with release signing (signing keys documented in team password manager)
- [ ] MDM configuration documentation written: `docs/mdm_configuration.md` specifies how to pre-configure all `--dart-define` values as a JSON configuration profile
- [ ] The `release.yml` GitHub Actions workflow is updated to build and output both packages on version tags
- [ ] iOS MDM: documented approach for deploying via JAMF or equivalent (upload `.ipa` to MDM, create deployment group)
- [ ] Android MDM: documented approach for deploying via `.apk` side-load or Managed Google Play private track

---

## Sub-tasks

- [ ] Update `.github/workflows/release.yml` to actually build the release:
  - iOS job on `macos-latest`: build with `flutter build ipa --release` (requires enterprise cert in secrets)
  - Android job on `ubuntu-latest`: build with `flutter build apk --release` (requires keystore in secrets)
  - Upload `.ipa` and `.apk` as workflow artifacts
- [ ] Create signing key infrastructure:
  - iOS: generate/obtain Apple Enterprise Distribution certificate; store as base64-encoded secret `IOS_CERTIFICATE_BASE64`
  - Android: generate keystore; store as `ANDROID_KEYSTORE_BASE64` + `ANDROID_KEY_ALIAS` + `ANDROID_KEY_PASSWORD` secrets
- [ ] Create `docs/mdm_configuration.md` specifying:
  - For JAMF: `Configuration Profile` with `Managed App Configuration` dictionary containing `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `ISS_POSITION_API` keys
  - Note: `--dart-define` values are compiled into the binary, so runtime MDM config is not possible. Instead, document building a custom binary for each operator with their values baked in.
  - Alternative: expose a runtime config file at a well-known path (e.g., `$DOCUMENTS/config.json`) that overrides built-in defaults; implement this if operator flexibility is required
- [ ] Test the `.apk` via `adb install what_on_earth.apk` on a device not enrolled in any development profile
- [ ] Test the `.ipa` via JAMF or Apple Configurator 2 if available; otherwise via ad-hoc profile
