# WOE-040: Pass Calculator Bridge Integration

**ID:** WOE-040
**Phase:** 5 — Pins
**Step:** 5.7
**Depends on:** WOE-011, WOE-035

---

## Description

Implement the `REQUEST_PASS_CALC` / `PASS_CALC_RESULT` bridge message pair that uses `satellite.js` inside the CesiumJS WebView to compute the next overhead pass for a given geographic location. This is called when the user opens a Pin Detail and when the Pin List screen needs pass countdowns.

**Reference:** TECH_SPEC §7.4, §8.1, §8.2

---

## Definition of Done

- [ ] `REQUEST_PASS_CALC` Flutter→JS message sends `{ requestId, lat, lon }`
- [ ] JS computes the next pass using `satellite.js` (iterates forward in 1-minute steps from now, max 48 hours, finds when elevation > 10°)
- [ ] `PASS_CALC_RESULT` JS→Flutter returns `{ requestId, passStartUtc, maxElevationDeg, passDurationSeconds }` or `{ requestId, error: 'no_pass_found' }` if no pass within 48 hours
- [ ] Flutter `PassCalculator` class wraps the bridge round-trip in a `Future<PassResult>` using the `requestId` to match responses
- [ ] Calculation uses the current TLE (set via `SET_TLE` in WOE-011) — fails gracefully if no TLE is loaded
- [ ] A `passTimesProvider` Riverpod provider caches pass times per pin ID and refreshes them every 30 minutes
- [ ] Unit test (Dart side): verify `requestId` matching works correctly for concurrent requests

---

## Sub-tasks

- [ ] In `web_globe/src/satellite_propagator.js`, implement `calculateNextPass(lat, lon, tleLine1, tleLine2)`:
  - Iterate from now to now + 48h in 60-second steps
  - At each step, propagate ISS position, compute observer range and elevation angle using `satellite.ecfToLookAngles(observerGd, positionEcf)`
  - Find the first step where elevation > 10°; track max elevation; find when elevation drops below 10° again for duration
  - Return `{ passStartUtc, maxElevationDeg, passDurationSeconds }` or null
- [ ] In `main.js`, handle `REQUEST_PASS_CALC`: call `calculateNextPass(payload.lat, payload.lon, currentTleLine1, currentTleLine2)` then call `window.flutter_inappwebview.callHandler('PASS_CALC_RESULT', { requestId: payload.requestId, ...result })`
- [ ] In Flutter `lib/pins/pass_calculator.dart`:
  - `Future<PassResult?> calculateNextPass(double lat, double lon)`: generate a `requestId` (UUID), send `REQUEST_PASS_CALC`, register a one-shot `Completer` keyed by `requestId` in a map, resolve it when `PASS_CALC_RESULT` arrives
  - Timeout after 10 seconds
- [ ] Create `PassResult` model: `passStartUtc` (DateTime), `maxElevationDeg` (double), `passDurationSeconds` (int)
- [ ] Create `passTimesProvider` that fetches pass times for all current pins on init and every 30 minutes
- [ ] Unit test for `requestId` concurrency
