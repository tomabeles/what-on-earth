# WOE-073: Telemetry HUD -- Roll Indicator

**ID:** WOE-073
**Phase:** UI -- Telemetry HUD
**Depends on:** WOE-070
**UI Spec:** SS5.2

---

## Description

Add the roll indicator arc above the heading tape at the top-center of the HUD. Shows a +/-60 degree visible arc with a moving pointer indicating current roll. Pointer color changes to `hudWarning` when |roll| > 30 degrees.

Uses mock roll data until sensor fusion is available.

---

## Definition of Done

- [ ] Roll indicator arc renders above the heading tape
- [ ] Visible range: +/-60 degrees
- [ ] Tick marks at 0, +/-10, +/-20, +/-30, +/-45, +/-60 degrees
- [ ] Moving pointer shows current `rollDeg`
- [ ] Pointer color: `hudPrimary` normally; `hudWarning` when `|rollDeg| > 30`
- [ ] Accepts `double rollDeg` parameter (-180 to +180)
- [ ] Test: pointer color changes at threshold

---

## Sub-tasks

- [ ] Add `paintRollIndicator(Canvas, Size, double rollDeg)` to `HudPainter`
- [ ] Draw arc: `canvas.drawArc` centered at top-center, spanning 120 degrees (60 per side), open downward
- [ ] Arc radius: ~50dp; positioned above heading tape area
- [ ] Tick marks: draw small radial lines at each tick position (0, +/-10, +/-20, +/-30, +/-45, +/-60)
- [ ] 0-degree tick slightly longer than others
- [ ] Pointer: filled triangle or line at the position corresponding to `rollDeg`, clamped to +/-60 visible range
- [ ] Color logic: `rollDeg.abs() > 30 ? tokens.hudWarning : tokens.hudPrimary`
- [ ] Use `Paint()..color = pointerColor..strokeWidth = 2.0`
- [ ] Add `rollDeg` to `HudPainter` constructor
- [ ] Mock: use static roll value for testing; try values at 0, 25, 45 to verify color change
- [ ] Test: rollDeg=25 uses hudPrimary; rollDeg=35 uses hudWarning
