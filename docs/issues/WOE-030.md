# WOE-030: TileDownloader Implementation

**ID:** WOE-030
**Phase:** 4 — Map Layers
**Step:** 4.6
**Depends on:** WOE-028

---

## Description

Implement the `TileDownloader` class that fetches raster tiles from remote tile servers and stores them in the app's documents directory in the structure expected by the tile server. Downloads are parallelized with 4 concurrent workers and progress is reported via a stream for the onboarding UI.

**Reference:** TECH_SPEC §7.3

---

## Definition of Done

- [ ] `TileDownloader` exists in `lib/tile_cache/tile_downloader.dart`
- [ ] `downloadLayer(layerId, sourceUrl, zoomRange)` downloads all tiles for a layer at the specified zoom levels and saves to `{documents}/tiles/{layerId}/{z}/{x}/{y}.ext`
- [ ] Concurrent download limit is 4 (configurable)
- [ ] Download progress is reported as a `Stream<TileDownloadProgress>` with `completedTiles`, `totalTiles`, `bytesDownloaded`
- [ ] If a tile already exists on disk and is not stale, it is skipped
- [ ] Download can be cancelled via a `CancelToken`
- [ ] Network errors on individual tiles are retried once then skipped (not fatal to the overall download)
- [ ] Unit tests with mocked `dio` and a temp directory

---

## Sub-tasks

- [ ] Create `lib/tile_cache/tile_downloader.dart`
- [ ] Implement `List<TileCoordinate> enumerateTiles(int minZoom, int maxZoom)`: enumerate all valid `{z}/{x}/{y}` combos for a Web Mercator tileset from zoom `minZoom` to `maxZoom`. At zoom 0: 1 tile; zoom 1: 4; zoom 2: 16; zoom 3: 64; zoom 4: 256; zoom 5: 1024 → total ~1365 tiles per layer for zoom 0–5
- [ ] Implement the download loop using `dio` with concurrency managed by a semaphore (use `async` package's `Pool` or implement with `Stream` and `asyncMap`)
- [ ] On each tile: construct the remote URL from the template, download bytes, write to `{documents}/tiles/{layerId}/{z}/{x}/{y}.webp` (or `.png`)
- [ ] Emit `TileDownloadProgress` events via `StreamController`
- [ ] `CancelToken` support: check cancellation between tiles
- [ ] Skip tiles where file exists and `downloadedAt` is within TTL (read from `TileCacheManager` metadata in WOE-031; for now, check if file exists on disk)
- [ ] Retry logic: on HTTP error, retry once after 1 second; if second attempt fails, log and skip
- [ ] Unit test: verify all tiles in zoom range 0–2 are attempted; verify cancellation stops the loop
