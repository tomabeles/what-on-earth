# WOE-055: Dark and Light Mode Verification

**ID:** WOE-055
**Phase:** 7 — NFR Pass and Polish
**Step:** 7.6
**Depends on:** WOE-049

---

## Description

Verify all Flutter UI screens in both dark mode (primary) and light mode (optional, per TECH_SPEC §10.4). Globe label colors in CesiumJS must be readable against both light and dark sky/globe backgrounds.

**Reference:** TECH_SPEC §10.4, PRD §10.4

---

## Definition of Done

- [ ] Dark mode: all screens render correctly; text is legible; no white elements on light backgrounds; globe labels readable against dark space
- [ ] Light mode: all screens render correctly; no dark text on dark backgrounds; globe labels readable against brighter globe surface colors
- [ ] The app respects the device's system dark/light mode preference by default
- [ ] A "Force dark / light mode" override is available in Settings
- [ ] Globe annotation label colors adapt to the current theme (white labels in dark mode, dark labels in light mode) via `THEME_CHANGE` bridge message

---

## Sub-tasks

- [ ] Ensure `MaterialApp.theme` and `MaterialApp.darkTheme` are both defined in `app.dart`; `themeMode: ThemeMode.system` (respects OS preference)
- [ ] Add a `themeMode` setting in Settings screen: radio buttons for "System", "Dark", "Light"; persist to shared preferences
- [ ] Review each screen in both themes using Flutter widget test with `MediaQuery` `platformBrightness` override:
  - `ARView` — status bar, layer panel, banners
  - `PinListScreen` — list tiles, empty state
  - `PinDetailScreen` — pass info, buttons
  - `OnboardingFlow` — all three steps
  - `SettingsScreen` — all sections
- [ ] Add `THEME_CHANGE` bridge message (Flutter → JS) with `{ isDark: bool }`. In CesiumJS: set label fill colors `Cesium.Color.WHITE` (dark mode) or `Cesium.Color.fromCssColorString('#222222')` (light mode); set border/coastline colors accordingly
- [ ] Send `THEME_CHANGE` on app launch (with current theme) and whenever theme changes
- [ ] Run widget tests for both themes; fix any color issues found
