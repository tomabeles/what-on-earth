# WOE-047: Magnetometer Calibration Routine Implementation

**ID:** WOE-047
**Phase:** 6 — Cloud Sync and Onboarding
**Step:** 6.5
**Depends on:** WOE-020

---

## Description

Implement the magnetometer calibration routine that guides the user through a figure-8 motion, collects raw magnetometer samples, computes hard-iron bias, and stores the calibration parameters. This is both used in the onboarding flow (step 3) and accessible from the Settings screen at any time.

**Reference:** TECH_SPEC §7.2, §9.5, PRD FR-SEN-003

---

## Definition of Done

- [ ] `CalibrationScreen` / `CalibrationStep` widget exists in `lib/onboarding/calibration_screen.dart`
- [ ] Animated figure-8 diagram guides the user through the required device motion
- [ ] Collects magnetometer samples for 15 seconds during the motion (or until "Done" is tapped)
- [ ] Computes hard-iron bias as the midpoint of the min/max range for each axis: `hard_iron[i] = (max[i] + min[i]) / 2`
- [ ] Displays a live accuracy indicator (simple bar or numeric) based on the spread of samples relative to an ideal sphere
- [ ] "Done" button is enabled when confidence > 80% (at least 100 samples collected with reasonable spread)
- [ ] On "Done": calls `CalibrationStore.save(params)` and `SensorFusionEngine.updateCalibration(params)`
- [ ] "Skip" button available at all times (with a warning that orientation accuracy may be reduced)
- [ ] Unit test for the hard-iron computation algorithm

---

## Sub-tasks

- [ ] Create `lib/onboarding/calibration_screen.dart`
- [ ] Animated figure-8: use a `CustomPainter` or a Flutter animation that shows a stylized phone icon tracing a figure-8 path in a loop
- [ ] Sample collection: subscribe to `sensors_plus` `magnetometerEventStream()` for 15 seconds; collect into a list of `Vector3` samples
- [ ] Compute hard-iron bias: for each axis x/y/z: `min = samples.map((s) => s.x).min`; `max = ...max`; `bias = (min + max) / 2`
- [ ] Confidence metric: compute the ratio of sample spread (max−min per axis) to expected spread (~60–70 µT typical Earth field); if spread is < 10 µT on any axis, confidence is low (user likely didn't do the figure-8 fully)
- [ ] Real-time accuracy indicator: a `LinearProgressIndicator` or animated bar that fills as confidence increases
- [ ] "Done" enabled when `confidence >= 0.8` and sample count >= 100
- [ ] On "Done": construct `CalibrationParams`, call `calibrationStore.save(params)`, call `ref.read(sensorFusionEngineProvider).updateCalibration(params)`, pop or advance to next onboarding page
- [ ] Unit test: `computeHardIron(samples)` returns the correct bias for known synthetic data
