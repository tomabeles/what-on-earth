# WOE-021: Magnetometer Interference Detection and UI Banner

**ID:** WOE-021
**Phase:** 3 — Sensor Fusion and AR Camera
**Step:** 3.5
**Depends on:** WOE-018, WOE-020

---

## Description

Implement interference detection in the sensor fusion engine and display a non-blocking banner in the AR view when interference is detected, prompting the user to recalibrate.

Per TECH_SPEC §7.2: if the magnetometer heading changes by more than 30° between consecutive samples at 50 Hz (implying >1500°/s — physically impossible from device motion), the engine sets `reliable: false` on the `DeviceOrientation` output and emits a `MagnetometerInterferenceEvent`.

**Reference:** TECH_SPEC §7.2, PRD FR-SEN-005

---

## Definition of Done

- [ ] When heading delta between consecutive samples exceeds 30°, `DeviceOrientation.reliable` is set to `false`
- [ ] A `MagnetometerInterferenceEvent` stream is exposed from `SensorFusionEngine`
- [ ] A dismissible banner "Compass interference detected — tap to recalibrate" appears on the AR view when the event fires
- [ ] Banner links to the calibration screen (WOE-047; stub navigation acceptable here)
- [ ] Banner auto-dismisses after 30 seconds if not tapped
- [ ] `reliable` returns to `true` after 5 consecutive stable readings
- [ ] Unit test: verify the threshold logic with synthetic heading sequences

---

## Sub-tasks

- [ ] In the sensor fusion isolate, track previous heading; if `|currentHeading - prevHeading| > 30° && headingDelta < 330°` (accounting for wrap-around), mark as interference
- [ ] Emit `reliable: false` on `DeviceOrientation` for the duration of interference
- [ ] Reset to `reliable: true` after 5 consecutive non-interfering samples
- [ ] Expose `Stream<MagnetometerInterferenceEvent> get interferenceEvents` from `SensorFusionEngine`
- [ ] Create `MagnetometerInterferenceEvent` model (simple event class, no payload needed)
- [ ] In AR view widget, listen to `interferenceEvents`; show a `MaterialBanner` or custom `AnimatedContainer` banner at the top of the screen
- [ ] Banner has a "Recalibrate" action button (navigation stubbed for now)
- [ ] Auto-dismiss timer: `Timer(Duration(seconds: 30), banner.dismiss)`
- [ ] Unit test for the 30°/5-sample threshold logic
