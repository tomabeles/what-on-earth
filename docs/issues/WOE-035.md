# WOE-035: PinRepository CRUD Implementation

**ID:** WOE-035
**Phase:** 5 — Pins
**Step:** 5.2
**Depends on:** WOE-034

---

## Description

Implement the `PinRepository` class that provides a clean domain-level API for creating, reading, updating, and soft-deleting pins. This wraps the raw drift DAO and maps between the database row type and the domain `Pin` model.

**Reference:** TECH_SPEC §7.4, §9.3

---

## Definition of Done

- [ ] `PinRepository` exists in `lib/pins/pin_repository.dart`
- [ ] `createPin(name, latDeg, lonDeg, iconId, note?)` creates a new `Pin` with a generated UUID, sets `createdAt` and `updatedAt` to now
- [ ] `watchAllPins()` returns `Stream<List<Pin>>` of all non-deleted pins, reactive to changes
- [ ] `updatePin(Pin updatedPin)` sets `updatedAt` to now and saves
- [ ] `deletePin(String id)` sets `deletedAt` to now (soft delete)
- [ ] `getPinById(String id)` returns a single pin or null
- [ ] All methods use the `AppDatabase` via the drift DAO
- [ ] Unit tests covering all CRUD operations on an in-memory database
