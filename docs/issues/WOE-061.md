# WOE-061: AR Screen Scaffold

**ID:** WOE-061
**Phase:** 1 — Foundation
**Step:** 1.7
**Depends on:** WOE-006, WOE-060

---

## Description

Replace the `ar_screen.dart` placeholder with a real composited AR screen: a `Stack` with a live camera preview at the bottom and the transparent CesiumJS globe on top. Wire `StaticPositionSource` (WOE-060) into the bridge so that the globe shows a satellite marker at the static position immediately on launch — no network required.

This is the first issue where you can launch the app on a real device or iOS Simulator and see something meaningful end-to-end: camera feed (or black fallback on Simulator) + globe + satellite dot moving on a timer.

Camera permission is requested at screen entry; a clear permission-denied UI is shown if the user declines rather than crashing or showing a blank screen.

**Reference:** TECH_SPEC §3.1 (stack layout), §3.2 (transparency), §8.1 (UPDATE_POSITION payload)

---

## Definition of Done

- [ ] `lib/screens/ar_screen.dart` is a real widget (not a placeholder)
- [ ] `Stack` layout: `CameraPreview` (full-screen, bottom) → `GlobeView` (full-screen, top, transparent background)
- [ ] Camera permission is requested via `permission_handler` on `initState`; if denied, the camera layer shows a solid black `Container` with a brief explanatory text — the globe still renders above it
- [ ] On Simulator (no camera hardware), the app does not crash; the camera layer gracefully degrades to the black fallback
- [ ] `StaticPositionSource` is started in `initState` and stopped in `dispose`; every emission calls `bridgeController.send(OutboundMessage.updatePosition, position.toJson())`
- [ ] `app.dart` routes `home` to `ARScreen` instead of directly to `GlobeView`
- [ ] `GlobeView` is made injectable / accepts an external `BridgeController` so tests can pass a mock
- [ ] `flutter analyze` passes on all modified files

---

## Sub-tasks

- [ ] Implement `lib/screens/ar_screen.dart`:
  - `StatefulWidget` with `_ARScreenState`
  - Fields: `CameraController? _camera`, `StaticPositionSource _positionSource`, `BridgeController _bridge`
  - `initState`: request camera permission → on granted, initialize `CameraController(cameras.first, ResolutionPreset.high)` and `await controller.initialize()`; on denied or Simulator failure, set `_camera = null` and continue
  - Start `_positionSource`, subscribe to `positionStream`, call `_bridge.send(OutboundMessage.updatePosition, pos.toJson())` on each emission
  - `dispose`: stop `_positionSource`, dispose `_camera`, dispose `_bridge`
  - `build`: `Stack(children: [_buildCamera(), GlobeView(bridge: _bridge)])`
  - `_buildCamera()`: if `_camera != null && _camera!.value.isInitialized` return `CameraPreview(_camera!)` sized to fill; otherwise return black `Container`
- [ ] Update `GlobeView` to accept an optional `BridgeController bridge` parameter (falling back to an internally created one if null, for backwards-compat)
- [ ] Update `lib/app.dart`: change `home` from `GlobeView()` to `ARScreen()`
- [ ] Add `camera` and `permission_handler` to `pubspec.yaml` if not already present
- [ ] Add required `Info.plist` key (`NSCameraUsageDescription`) for iOS and `<uses-permission>` for Android camera if not already present (check existing scaffold from WOE-001)
- [ ] Manual test checklist (add to PR description):
  - [ ] Launch on iOS Simulator: globe loads, satellite dot visible, no crash
  - [ ] Launch on physical iOS device: camera feed visible beneath globe, satellite dot visible
  - [ ] Deny camera permission on physical device: black background + message shown, globe still renders
