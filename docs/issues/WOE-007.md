# WOE-007: GitHub Actions CI Pipeline

**ID:** WOE-007
**Phase:** 1 — Foundation
**Step:** 1.7
**Depends on:** WOE-001, WOE-002

---

## Description

Create the GitHub Actions CI pipeline that runs on every pull request. The pipeline must: install the correct toolchain versions, build the CesiumJS bundle, run Flutter code generation, lint, test, and smoke-build for both Android and iOS.

A separate `release.yml` workflow is also created (skeleton only in this issue — actual signing configuration is deferred to WOE-056) that runs on version tags.

**Reference:** TECH_SPEC §6.5

---

## Definition of Done

- [ ] `.github/workflows/ci.yml` exists and passes on the current `main` branch
- [ ] CI installs: Flutter 3.27.x, Node 22 LTS
- [ ] CI runs `scripts/build_globe.sh` (CesiumJS Vite build) before any Flutter steps
- [ ] CI runs `flutter pub get`
- [ ] CI runs `dart run build_runner build --delete-conflicting-outputs`
- [ ] CI runs `flutter analyze` and fails the build on any lint error
- [ ] CI runs `flutter test` and fails on any test failure
- [ ] CI runs `flutter build apk --debug` and `flutter build ios --no-codesign --debug`
- [ ] `.github/workflows/release.yml` exists as a skeleton triggered on `v*` tags (build steps stubbed with `echo "TODO"`)
- [ ] CI completes in under 15 minutes on a standard GitHub-hosted runner

---

## Sub-tasks

- [ ] Create `.github/workflows/ci.yml`:
  - Trigger: `on: [pull_request, push to main]`
  - Job: `build` on `ubuntu-latest`
  - Step: checkout with `fetch-depth: 0`
  - Step: setup Node 22 using `actions/setup-node@v4`
  - Step: setup Flutter 3.27.x using `subosito/flutter-action@v2` with `channel: stable`
  - Step: `cd web_globe && npm ci && npx vite build --outDir ../assets/globe`
  - Step: `flutter pub get`
  - Step: `dart run build_runner build --delete-conflicting-outputs`
  - Step: `flutter analyze`
  - Step: `flutter test`
  - Step: `flutter build apk --debug` (Android smoke build)
  - Add a separate job `build-ios` on `macos-latest` for the iOS smoke build: `flutter build ios --no-codesign --debug`
- [ ] Cache: add `actions/cache@v4` for `~/.pub-cache` (Dart packages) and `web_globe/node_modules`
- [ ] Create `.github/workflows/release.yml`:
  - Trigger: `on: push: tags: ['v*']`
  - Job: `release` on `ubuntu-latest`
  - All steps: `echo "TODO: configure signing and upload"`
- [ ] Verify CI passes on the current state of the repo by pushing to a test branch
