# WOE-083: Settings -- Account, Sensor, Power, About Sections

**ID:** WOE-083
**Phase:** UI -- Settings
**Depends on:** WOE-045, WOE-084, WOE-051
**UI Spec:** SS4.6

---

## Description

Add the remaining four sections to the Settings screen: Account, Sensor, Power, and About. These sections have the most backend dependencies and are implemented last.

The About section has no backend dependencies and can be built immediately as a sub-task. Other sections depend on Supabase auth (Account), calibration (Sensor), and power-saving mode (Power).

---

## Definition of Done

- [ ] **Account section:** "Sign In" button (when not authenticated) or signed-in email + "Sign Out" button; sync status ("Last synced X ago")
- [ ] **Sensor section:** "Recalibrate magnetometer" button navigating to Calibration screen; last calibration date and confidence %
- [ ] **Power section:** Power-saving mode toggle; threshold slider (5-50%, default 20%)
- [ ] **About section:** App version; Open Data Registry (scrollable list of data sources with license); open-source licenses link; data attribution
- [ ] All sections render even when backend dependencies are unavailable (show appropriate fallback state)
- [ ] Widget test: all sections render; About section always visible

---

## Sub-tasks

- [ ] **Account section card:**
  - Watch Supabase auth state; show email + "Sign Out" if authenticated, "Sign In" button if not
  - "Sign In" navigates to auth flow (WOE-045)
  - Sync status: read last sync timestamp from PinSync; format as relative time
  - If auth not available: show "Sign in to sync pins across devices"
- [ ] **Sensor section card:**
  - "Recalibrate magnetometer" `ListTile` with trailing arrow; onTap: `Navigator.push(CalibrationScreen)`
  - Subtitle: "Last calibrated: [date]" + "Confidence: [X]%" from `CalibrationStore`
  - If no calibration data: "Not calibrated"
- [ ] **Power section card:**
  - `SwitchListTile` for "Power-saving mode" (reduces sensor + frame rates)
  - When enabled: show `Slider` for battery threshold (5-50%, default 20%)
  - Persist both values in SharedPreferences
- [ ] **About section card:**
  - App version: `PackageInfo.fromPlatform().then((info) => info.version)`
  - Data Registry: `ExpansionTile` listing: OpenStreetMap (ODbL), Natural Earth (public domain), CelesTrak (public domain), wheretheiss.at (attribution), satellite.js (MIT)
  - "Open-source licenses": `ListTile` that opens `LicensePage()`
- [ ] Widget test: render settings screen, verify all 4 sections exist with correct titles
