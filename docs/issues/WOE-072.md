# WOE-072: Telemetry HUD -- Pitch Ladder

**ID:** WOE-072
**Phase:** UI -- Telemetry HUD
**Depends on:** WOE-070
**UI Spec:** SS5.2

---

## Description

Add the pitch ladder to the left side of the HUD. Horizontal lines at +/-5 degree increments from the current pitch, labeled at +/-10, +/-20, +/-30 degrees. Lines shorten toward center.

Uses mock pitch data until sensor fusion is available.

---

## Definition of Done

- [ ] Pitch ladder renders on the left portion of the HUD
- [ ] Lines at +/-5 degree increments relative to current pitch
- [ ] Labels at +/-10, +/-20, +/-30 degrees
- [ ] Line length decreases toward center: +/-30 lines are longest, +/-5 are shortest
- [ ] Color: `hudPrimary` at 60% opacity
- [ ] Accepts `double pitchDeg` parameter (-90 to +90)
- [ ] Test: verify correct number of lines for a given pitch value

---

## Sub-tasks

- [ ] Add `paintPitchLadder(Canvas, Size, double pitchDeg)` to `HudPainter`
- [ ] Define pitch increments: +/-5, +/-10, +/-15, +/-20, +/-25, +/-30 from current pitch
- [ ] Calculate Y position for each increment: map degree offset to vertical pixels (e.g., 4dp per degree)
- [ ] Line lengths: `|increment| / 30 * maxLength` where `maxLength` is ~80dp; minimum length ~20dp for +/-5
- [ ] Draw each line centered horizontally on the left third of screen (X offset ~60dp from left edge)
- [ ] Label lines at +/-10, +/-20, +/-30 with degree text (e.g., "+20") using `hudFontFamily` at `hudFontSize - 2`
- [ ] Use `Paint()..color = tokens.hudPrimary.withOpacity(0.6)..strokeWidth = 1.0`
- [ ] Add `pitchDeg` to `HudPainter` constructor
- [ ] Mock: use static pitch value (e.g., 12.0) for visual testing
- [ ] Test: render with pitchDeg=0, verify symmetric lines above and below center
