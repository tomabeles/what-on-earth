# WOE-026: CesiumJS GeoJSON Layer Rendering (layers.js)

**ID:** WOE-026
**Phase:** 4 — Map Layers
**Step:** 4.2
**Depends on:** WOE-025, WOE-004

---

## Description

Implement `layers.js` in the CesiumJS bundle to load and style the Natural Earth GeoJSON assets as CesiumJS `GeoJsonDataSource` layers. The four vector layers (country borders, coastlines, lakes, city points) must render at orbital zoom with high-contrast styling appropriate for a dark background.

GeoJSON files are fetched from `http://localhost:8080/geodata/{filename}` (served by `InAppLocalhostServer` from Flutter assets).

**Reference:** TECH_SPEC §3.5, §4.5

---

## Definition of Done

- [ ] Country borders render as white polylines (0.7 opacity) on the globe at orbital altitude
- [ ] Coastlines render as light blue polylines
- [ ] Lakes render as semi-transparent blue polygons
- [ ] City points render as white dots with city name labels; label size scales appropriately at orbital zoom; only cities with `POP_MAX > 500000` show labels (to avoid clutter)
- [ ] All four layers load successfully from the local asset server
- [ ] Layers are stored in a `layersRegistry` object keyed by layer ID so they can be toggled by WOE-027
- [ ] No console errors or CesiumJS warnings during layer load

---

## Sub-tasks

- [ ] Create `web_globe/src/layers.js`
- [ ] Define `layersRegistry = {}` object
- [ ] Implement `async function loadVectorLayers(viewer)`:
  - Load each GeoJSON via `Cesium.GeoJsonDataSource.load('http://localhost:8080/geodata/{filename}')` where port matches `CESIUM_LOCALHOST_PORT`
  - Style country borders: `stroke: Cesium.Color.WHITE.withAlpha(0.7)`, `strokeWidth: 1`, `fill: Cesium.Color.TRANSPARENT`
  - Style coastlines: `stroke: Cesium.Color.fromCssColorString('#6ab0cc')`, `strokeWidth: 1.5`
  - Style lakes: `fill: Cesium.Color.fromCssColorString('#1a4a6b').withAlpha(0.6)`, no stroke
  - Style city points: `markerSize: 4`, `markerColor: Cesium.Color.WHITE`; add label on `name` property; set `label.show = entity.properties.POP_MAX.getValue() > 500000`
  - Add each `GeoJsonDataSource` to the viewer: `viewer.dataSources.add(ds)`
  - Store in `layersRegistry['borders']`, `layersRegistry['coastlines']`, etc.
- [ ] Call `loadVectorLayers(viewer)` in `main.js` after `GLOBE_READY` message setup
- [ ] Test on device: verify layers are visible when globe is positioned over land
