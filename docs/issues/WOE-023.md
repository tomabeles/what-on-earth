# WOE-023: ARView Stack Widget Assembly

**ID:** WOE-023
**Phase:** 3 — Sensor Fusion and AR Camera
**Step:** 3.7
**Depends on:** WOE-004, WOE-022, WOE-019

---

## Description

Assemble the `ARView` widget that combines the camera preview background and the transparent CesiumJS WebView overlay into the complete AR experience. Also layer the Flutter UI chrome (status bar, layer toggles) on top. This is the primary screen of the app.

**Reference:** TECH_SPEC §3.1, §10.2

---

## Definition of Done

- [ ] `ARView` widget exists in `lib/globe/ar_view.dart`
- [ ] Renders as a `Stack` with three layers (bottom to top):
  1. `CameraPreviewLayer` — full-screen camera feed
  2. `GlobeView` — transparent CesiumJS WebView
  3. Flutter UI chrome — `StatusBar`, layer toggle panel (stub), navigation buttons
- [ ] The globe is visible overlaid on the camera feed
- [ ] Status bar is visible and functional (from WOE-015)
- [ ] All three layers are sized to fill the screen
- [ ] Navigation: a floating action button or bottom nav allows switching to the 2D Map screen and Pin List screen (screens can be stubs at this point)

---

## Sub-tasks

- [ ] Create `lib/globe/ar_view.dart` with a `Stack`:
  - Layer 1: `CameraPreviewLayer()` wrapped in `Positioned.fill`
  - Layer 2: `GlobeView()` wrapped in `Positioned.fill`
  - Layer 3: `Positioned(top: 0, left: 0, right: 0, child: StatusBar())` from WOE-015
  - Layer 3: `Positioned(bottom: 16, right: 16, child: _LayerToggleButton())` (stub)
  - Layer 3: `Positioned(bottom: 16, left: 16, child: _NavigationControls())` (stub)
- [ ] Set `SystemChrome.setEnabledSystemUIMode(SystemUiMode.immersiveSticky)` in `ARView.initState` to use the full screen
- [ ] Wire `ARView` as the home screen in `app.dart`
- [ ] Manual test: verify camera + globe are visible simultaneously on both iOS and Android
