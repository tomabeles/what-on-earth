# WOE-062: Integration Test Harness

**ID:** WOE-062
**Phase:** 1 — Foundation
**Step:** 1.8
**Depends on:** WOE-061

---

## Description

Set up the `integration_test` package and write a smoke test that runs end-to-end on an iOS Simulator (or Android emulator) as part of the GitHub Actions pipeline (WOE-007). The test pumps the real app, waits for the globe to signal readiness, and asserts that an `UPDATE_POSITION` message was dispatched to CesiumJS — verifying the full stack from app launch through bridge communication without touching any network.

Integration tests run against the real Flutter engine on a real platform (not the widget test sandbox), so `flutter_inappwebview` and `CameraPreview` behave as they do in production. The `StaticPositionSource` from WOE-060 provides deterministic, offline position data.

**Reference:** TECH_SPEC §6.3 (testing strategy)

---

## Definition of Done

- [ ] `integration_test/` directory exists with a `flutter_test_config.dart` and at least one test file
- [ ] `integration_test/smoke_test.dart` pumps `WhatOnEarthApp` and asserts:
  - The app launches without throwing (no red screen)
  - Within 20 seconds, a widget with key `Key('globe_view')` is present in the tree (globe loaded)
  - Within 20 seconds after `GLOBE_READY`, at least one `UPDATE_POSITION` JavaScript call has been dispatched (verified via an injected JS listener or a `ValueNotifier` exposed for testing)
- [ ] A dedicated `integration_test/helpers/test_app.dart` wraps `WhatOnEarthApp` in `ProviderScope` with overrides that substitute `StaticPositionSource` for any live or TLE source, so the test is fully offline
- [ ] `flutter test integration_test/smoke_test.dart -d <simulator-id>` passes locally on macOS
- [ ] GitHub Actions (`.github/workflows/ci.yml`) gains a new `integration-test` job:
  - Runs on `macos-latest`
  - Boots an iOS Simulator via `xcrun simctl`
  - Runs `scripts/build_globe.sh` then `flutter test integration_test/` targeting the booted Simulator
  - Uploads simulator screenshots on failure as a build artifact
- [ ] `flutter analyze` passes on all new files

---

## Sub-tasks

- [ ] Add `integration_test` (SDK package, no version needed) and `flutter_test` to `dev_dependencies` in `pubspec.yaml`
- [ ] Create `integration_test/flutter_test_config.dart` (empty `Future<void> testExecutable(FutureOr<void> Function() testMain) async => testMain()`)
- [ ] Create `integration_test/helpers/test_app.dart`:
  - Exports a `pumpTestApp(WidgetTester tester)` helper that wraps `WhatOnEarthApp` in `ProviderScope` with a `StaticPositionSource` override (interval: 1 second for test speed)
  - Stores a `List<Map<String, dynamic>> dispatchedPositions = []` that a test-only JS snippet appends to on each `UPDATE_POSITION` event
- [ ] Create `integration_test/smoke_test.dart`:
  - `testWidgets('app launches and globe renders', ...)`
  - Call `pumpTestApp(tester)`
  - `await tester.pumpAndSettle(timeout: const Duration(seconds: 20))`
  - Assert `find.byKey(const Key('globe_view')).findsOneWidget`
  - `testWidgets('UPDATE_POSITION dispatched after GLOBE_READY', ...)`
  - Call `pumpTestApp(tester)`, wait up to 20 s, assert `dispatchedPositions` is non-empty and first entry contains `lat`, `lon`, `altKm` keys
- [ ] Add `Key('globe_view')` to `GlobeView` widget in `lib/globe/globe_view.dart`
- [ ] Add `integration-test` job to `.github/workflows/ci.yml`:
  ```yaml
  integration-test:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.x'
          channel: 'stable'
      - run: flutter pub get
      - run: scripts/build_globe.sh
      - name: Boot iOS Simulator
        run: |
          UDID=$(xcrun simctl create "CI-iPhone" "iPhone 16" "iOS 18.0" 2>/dev/null || \
                 xcrun simctl list devices available --json | \
                 python3 -c "import sys,json; devs=[d for r in json.load(sys.stdin)['devices'].values() for d in r if 'iPhone' in d['name'] and d['isAvailable']]; print(devs[0]['udid'])")
          xcrun simctl boot "$UDID" || true
          echo "SIMULATOR_UDID=$UDID" >> $GITHUB_ENV
      - run: flutter test integration_test/ -d "$SIMULATOR_UDID" --reporter=expanded
      - name: Upload screenshots on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: simulator-screenshots
          path: "*.png"
  ```
