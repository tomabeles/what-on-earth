# WOE-078: 2D Map View Screen + SET_MODE Bridge Message

**ID:** WOE-078
**Phase:** UI -- Screens
**Depends on:** WOE-066, WOE-067
**UI Spec:** SS4.2

---

## Description

Implement the 2D Map View screen, replacing the placeholder from WOE-066. The screen reuses the same CesiumJS WebView but sends `SET_MODE {mode: 'map'}` to switch CesiumJS to a top-down 2D map camera mode. The user can pan and zoom freely. Status Bar is visible, Telemetry HUD is hidden, Controls button is visible (with camera row hidden), NAV FAB shows Map as active.

Also implement the `SET_MODE` bridge message and its CesiumJS handler.

---

## Definition of Done

- [ ] `MapScreen` in `lib/screens/map_screen.dart` (replace placeholder)
- [ ] Sends `SET_MODE {mode: 'map'}` on enter; sends `SET_MODE {mode: 'ar'}` on exit
- [ ] `SET_MODE` added to `OutboundMessage` in bridge.dart
- [ ] CesiumJS handler: `map` mode sets camera to top-down view with user-controlled pan/zoom; `ar` mode restores position-driven camera
- [ ] Status Bar visible at top
- [ ] Telemetry HUD NOT visible (hidden regardless of HUD toggle setting)
- [ ] Controls button visible; camera row hidden in Control Panel
- [ ] NAV FAB visible with Map button highlighted
- [ ] User can pan and zoom the 2D map freely
- [ ] Widget test: SET_MODE message sent on screen enter

## Sub-tasks

- [ ] In `bridge.dart`: add `setMode` to `OutboundMessage` with `messageName` `'SET_MODE'`
- [ ] Add `setMode(String mode)` convenience method to `BridgeController`
- [ ] In `web_globe/src/main.js`: add `SET_MODE` handler:
  - `'map'`: set `viewer.scene.mode = Cesium.SceneMode.SCENE2D` (or `COLUMBUS_VIEW`); enable default camera controls; zoom to show full globe
  - `'ar'`: set `viewer.scene.mode = Cesium.SceneMode.SCENE3D`; disable default camera input (orientation-driven)
- [ ] Update `lib/screens/map_screen.dart`:
  - `initState`: send `SET_MODE {mode: 'map'}`
  - `dispose`: send `SET_MODE {mode: 'ar'}`
  - Layout: `Stack` with globe view, Status Bar, Controls button (isMapMode: true), NAV FAB (activeDestination: 'map')
  - No `TelemetryHud` in the stack
- [ ] Run `scripts/build_globe.sh`
- [ ] Test: enter map screen, verify 2D view; exit, verify AR mode restored
